# IO HR 플랫폼 기능 개선 개발 계획서

> 미팅 녹취록 기반 기능 개선 요구사항 정리
> 대상 영역: 근태 관리 / 전자결재 / 채용 관리 / 연차·연장근로

---

## 0. 문서 개요

본 문서는 IO HR 플랫폼 운영 미팅에서 도출된 기능 개선 및 버그 수정 항목을 정리한 것이다.
크게 **① 근태·급여**, **② 전자결재**, **③ 채용 관리**, **④ 기타 UI/UX** 네 영역으로 나뉘며,
각 항목은 현재 상태 → 문제점 → 개선 방향 순으로 기술한다.

우선순위는 `🔴 긴급 (버그/미작동)` / `🟡 중요 (핵심 기능 미완)` / `🟢 개선 (UX 향상)` 으로 구분한다.

---

## 1. 근태 · 급여 관리

### 1.1 출퇴근 기본값 설정

**현재 상태**
- 출퇴근 기록을 하지 않은 직원에 대한 기본값 처리 로직 미정의

**개선 방향**
- 출퇴근 미기록 시 **09:00 ~ 18:00 (8시간)** 을 기본값으로 자동 처리
- 주 40시간 기준으로 설정
- 급여 계산의 기본 베이스: 월 고정 급여 → 연차 차감 → 연장근로 추가 순으로 계산

**세부 로직**
```
기본 급여
  - 연차 사용 시 → 해당 일수 차감
  - 연장근로 승인 완료 시 → 해당 시간 추가
  - 입사·퇴사일 기준 일할 계산 (현재 구현됨)
  - 퇴직금 + 연차수당 + 연장근로수당 자동 산출 (현재 구현됨)
```

---

### 1.2 연장근로 사전 승인제

**현재 상태**
- 연장근로 사전 승인 메뉴 구현되어 있으나 실제 작동 여부 미검증

**문제점**
- 직원들이 연장근무를 하고도 신청을 안 해 수당 미지급 → 노무 리스크 발생
- 18시 이후 연장 신청 알림이 직원에게 전달되는지 확인되지 않음

**개선 방향**
- **18:00 이후** 연장근로 중인 직원에게 **연장근무 신청 알림 자동 발송**
- 연장근무 신청 → 승인 완료 시에만 연장수당 카운트
- 미신청 연장근무는 수당 미지급 처리 (정책 명시 필요)
- **주 52시간 초과 시 시스템 블록** 처리 여부 확인 및 구현
  - 기준: 주 40시간 기본 + 연장 12시간 = 최대 52시간
  - 52시간 초과 시 추가 연장 신청 불가 처리

**검증 필요 항목**
- [ ] 18시 알림 실제 발송 확인
- [ ] 연장 승인 후 급여 카운터 반영 확인
- [ ] 52시간 초과 블록 작동 여부 확인

---

### 1.3 연차 촉진 자동화

**현재 상태**
- 연차 촉진 메뉴 존재 (ON/OFF 토글 방식)
- 2개월 전 / 6개월 전 촉진 설정되어 있으나 실제 발송 여부 미검증
- 테스트 시 아무 액션 없음 (메일/알림 미수신 확인됨)

**문제점**
- 연차 촉진이 제대로 작동하지 않으면 **연차수당 지급 의무 발생** → 법적 리스크
- 촉진 발송 후 직원이 **열람·확인해야 인정되는지**, 수신만으로 인정되는지 프로세스 미정의

**개선 방향**
- 연차 촉진 자동 발송 로직 검증 및 수정
- 발송 대상: 잔여 연차가 있는 직원 전체
- 발송 방식: 이메일 (추후 사내 메신저 연동 가능하도록 설계)
- **촉진 발송 이력 관리**: 발송 일시, 수신 확인 여부 기록
- 직원이 확인(열람) 시 확인 처리되는 플로우 구현

**연차 미사용 처리 플로우**
```
연차 촉진 발송 (2개월 전)
  → 직원 미사용 시 2차 촉진 발송 (1개월 전)
    → 미사용 시 연차 포기 각서 수령 프로세스 진입
      → 서명 완료 시 연차수당 미지급 처리
      → 미서명 시 연차수당 지급 의무 유지
```

### 1.4 연차 포기 각서 전자서명 🔴

**현재 상태**
- 연차 포기 각서 양식은 별도 존재 (이민지 담당자가 보유)
- 시스템 내 전자서명·관리 기능 없음

**개선 방향**
- 연차 내역 화면 내 **연차 포기 각서 전자서명 기능** 추가
- 서명 완료 시 연차수당 미지급 처리와 자동 연동
- 서명 이력 보관 (법적 증빙용)

---

### 1.5 연차 신청 후 출근 처리 케이스 🟡

**현재 상태**
- 연차 신청 후 실제로 출근하는 케이스 발생
- 수동으로 연차 취소 처리 중

**개선 방향**
- 연차 신청일에 출근 기록이 찍힐 경우 **자동으로 연차 취소 또는 알림** 처리
- 혹은 관리자 화면에서 연차 취소를 간편하게 처리할 수 있는 UI 제공

## 2. 전자결재

### 2.1 결재선 설정 오류 수정 🔴

**현재 상태**
- 휴가계 결재선 설정 시 **김보미 팀장이 결재자로 나타나지 않는** 오류 발생
- 강제묵·김영석·안병철로 결재선이 잘못 설정되어 있음

**확인된 원인**
- 역할을 "인사 담당자"로 변경하면서 결재선이 꼬인 것으로 추정

**개선 방향**
- 휴가계 결재선 수정: **김보미 팀장 → 임원 그룹(강제묵·김영석·안병철) → 대표** 순 복구
- 역할 변경 시 기존 결재선에 영향 없도록 로직 분리

---

### 2.2 결재선 단계 내 순서 처리 방식 개선 🟡

**현재 상태**
- 동일 단계 내 결재자가 순서대로만 결재 가능 (예: 이사 3명이 순서대로만 가능)

**요구사항**
- 결재선 구조:
  1. **1단계**: 김보미 팀장 (순서 고정)
  2. **2단계**: 강제묵 이사·김영석 이사·안병철 이사 → **순서 무관, 아무나 먼저 결재 가능**
  3. **3단계**: 대표 (순서 고정)

**개선 방향**
- 결재 단계 내 결재자 간 **병렬 결재 / 순차 결재 선택 옵션** 추가
- 2단계처럼 그룹 결재는 병렬(아무나 먼저)로 처리
- 일일 업무보고 등 전사 문서는 부서별로 결재선을 별도 구성하지 않아도 되도록 처리 (비효율 방지)

---

### 2.3 결재 대기 알림 개선 🟡

**현재 상태**
- 결재 요청 시 결재자에게 알림 미전달 또는 확인이 어려운 상태

**요구사항**
- 결재 요청 시 **이메일 알림** 발송 (가장 확실한 채널)
- 추가로 앱 푸시 알림 또는 사내 메신저 연동 고려
- 대시보드 상단에서 **"내가 결재할 항목"을 클릭 한 번으로 바로 진입** 가능하도록 구성
  - 현재: 대시보드 → 메뉴 진입 → 내 결재함 → 항목 선택 (多 단계)
  - 목표: 대시보드 위젯 클릭 → 즉시 결재 화면 진입

---

### 2.4 결재 목록 UI 개선 🟢

**현재 상태**
- 결재 목록에서 신청자 이름이 너무 작게 표시됨
- 모바일에서 이름이 안 보인다는 피드백

**개선 방향**
- 결재 목록에 **신청자 이름 폰트 사이즈 확대** (PC/모바일 공통)
- 1일 업무보고 목록에 날짜 외 **직원 이름 강조 표시** 추가

---

### 2.5 지출 결의서 UI 한국어화 🟢

**현재 상태**
- 지출 결의서 항목 내 일부 텍스트가 영문으로 표시됨 (예: Category, Bank 등)

**개선 방향**
- 지출 결의서 내 모든 레이블·카테고리명 **한국어로 전면 변경**

---

### 2.6 면접 평가 코멘트 기능 개선 🟡

**현재 상태**
- 면접 평가 화면 하단 코멘트 입력 가능
- 자동 저장 방식이나 저장 완료 여부를 알 수 없음
- 수정 기능 없음
- 코멘트가 면접관 간 공유용인지, 인사팀 전달용인지 구분 없음

**개선 방향**
- **저장 버튼 추가** (자동 저장 + 명시적 저장 버튼 병행)
- **코멘트 수정 기능** 추가
- 코멘트 유형 선택 기능 추가:
  - `면접관 간 공유 코멘트` (다른 면접관만 열람)
  - `인사팀 전달 메시지` (인사 담당자에게 알림 발송)
- **면접관 의견 선택 필드 추가**: 합격 / 불합격 / 보류
  - 실제 합격/불합격 처리와 연동되지 않음 (면접관 의견 단계)
  - 최종 결정은 인사 담당자 또는 이사가 별도 처리

## 3. 채용 관리

### 3.1 지원자 카드 — 유입 채널 및 공고 정보 표시 🟡

**현재 상태**
- 지원자 상세 화면에 유입 채널(알바몬, 사람인, 직접 지원 등) 표시 없음
- 어떤 채용 공고를 보고 지원했는지 직무 정보 미표시

**요구사항 (대표 요청)**
- 지원자 카드/상세에 **유입 채널** 표시 (예: 알바몬 / 사람인 / 직접 지원 / 헤드헌터)
- 지원 공고의 **직무 상세 정보** 연결하여 면접관이 확인 가능하도록
- 면접관 공유 링크 화면에도 동일하게 표시

**개선 방향**
- 지원자 등록 시 **출처(유입 채널) 필드** 추가 (필수 입력)
- 채용 공고와 지원자 연결 → 지원자 카드에 공고 직무 상세 자동 표시
- 직접 지원자의 경우 경로 불명확 → 입력 양식에 "직접 지원" 옵션 명시

---

### 3.2 사전 질문지(사전지리서) 응답 결과 통합 표시 🟡

**현재 상태**
- 사전 질문지 v2.0 사용 중이나, 응답 결과가 지원자 카드에 표시되지 않음
- 고령 지원자의 경우 구글 폼(구버전) 사용 → 시스템 외부에서 별도 관리
- 면접관이 사전 질문지 결과를 확인하려면 구글 폼에 별도 접속 필요

**개선 방향**
- 지원자 카드 내 **사전 질문지 응답 결과 통합 표시**
- 표시 항목 (요약 우선, 상세는 펼쳐보기):
  - 기본 정보 응답
  - 성향 분석 결과 요약
  - 적합도 점수
  - 주요 항목 요약
- **추가 문항 삽입** (대표 요청):
  - 가족관계
  - 희망 연봉
  - 출근 가능 일정
- 구버전(구글 폼) 응답은 PDF로 첨부하는 방식 유지 또는 수동 입력 필드 제공

---

### 3.3 포트폴리오 첨부 용량 정책 확립 🟢

**현재 상태**
- 현재 업로드 용량 제한: 50MB
- 디자이너 포트폴리오의 경우 1GB 이상인 경우도 있음

**개선 방향**
- 정책 결정: **50MB 이하 파일 업로드 or 클라우드 링크 입력** 방식으로 이원화
  - 50MB 초과 시 업로드 불가 안내 + 링크 입력 유도
  - 링크 입력 필드 별도 제공 (Google Drive, Notion, Behance 등)
- 파견 업체 채용 폼에도 동일 정책 적용

---

### 3.4 면접 녹화·스크립트 활용 가이드 🟢

**현재 상태**
- 화상 면접 시 녹화를 별도로 눌러야 하며, 누락되는 경우 발생
- 자동 스크립트 생성 기능이 있으나 면접관들이 인지 못함

**개선 방향**
- 화상 면접 진행 시 **스크립트 자동 생성 안내 문구** 표시
- 녹화 없이 스크립트만으로도 AI 분석 가능함을 면접관에게 고지
- 면접 완료 후 스크립트 + 회의록 요약을 지원자 카드에 자동 연동

---

### 3.5 채용 평가 권한 — 인사 담당자 역할 이슈 🔴

**현재 상태**
- 역할을 "인사 담당자"로 설정하면 **채용 평가 메뉴가 사라짐**
- 역할을 "시스템 관리자"로 설정해야만 평가 메뉴 접근 가능
- 두 가지 권한을 동시에 가질 수 없음

**개선 방향**
- **인사 담당자 역할에 채용 평가 접근 권한 추가**
- 역할별 메뉴 접근 권한 매트릭스 재설계:

| 메뉴 | 일반 직원 | 팀장 | 인사 담당자 | 임원 | 대표 |
|---|:---:|:---:|:---:|:---:|:---:|
| 채용 평가 | ✗ | ✗ | ✅ | ✅ | ✅ |
| 지원자 관리 | ✗ | ✗ | ✅ | ✅ | ✅ |
| 전자결재 | ✅ | ✅ | ✅ | ✅ | ✅ |
| 근태 관리 | ✅(본인) | ✅(팀) | ✅(전체) | ✅ | ✅ |

## 4. 기타 UI/UX 개선

### 4.1 업무 만족도 결재자 노출 🔴

**현재 상태**
- 업무 만족도 입력 화면이 결재자 계정에서 보이지 않음 (어제부터 발생)

**개선 방향**
- 결재자 계정에서 업무 만족도 화면 정상 노출되도록 수정

---

### 4.2 익명 신고 채널 활성화 가이드 🟢

**현재 상태**
- 익명 신고 기능 구현됨 (공지사항/게시글 형태)
- 활성화 방법 및 운영 가이드라인만 잡혀 있음

**개선 방향**
