## 1. 프로젝트 개요

INTEROHRIGIN 임직원 업무 관리 통합 플랫폼. 프로젝트 관리 · 전자결재 · 연차 · 증명서 · 근태 · 평가 · CEO 리포트를 단일 시스템으로 통합 운영.

### 핵심 설계 원칙
- **전사 일괄 오픈 ❌ → 경영지원팀 선행 테스트 → 단계별 확산**
- 키맨 중심 운영 → 사용성 익숙해지며 점진 확산
- CEO 리포트 한 방에 회사 전반을 파악할 수 있는 구조

### 기술 스택 / 인프라
- React + TypeScript + Tailwind CSS
- Supabase (DB / Auth)
- Cloudflare Pages (배포)
- Google Meet 자동 회의록 연동

### 2.2 📋 전자결재

#### 현재 이슈
- 양식별 결재선이 모두 상이 (일반결재 / 출장 / 지출결의 / 연차 등)
- 리더 부재 팀 존재 → 임원이 리더 대체
- 재무회계팀(2명) 전원 결재 필수 — 합의 개념 (순서 무관)
- 부서 / 본부별 결재 라인 다름

#### 🔧 개발 방향

##### (1) 결재선 수정 / 관리 기능
- [ ] 관리자(어드민 + 인사팀) 권한으로 **결재선 직접 수정 가능**
- [ ] 양식별 · 부서별 결재선 저장
- [ ] 기존 웍스 결재 관리 페이지 UI 참고 (민지님 캡처 전달 예정)

##### (2) 양식별 결재선 구조

**지출결의서**
```
담당자 → 리더 → 재무회계팀(2명 동시) → 관리자 → 임원
```

**일반결재**
```
리더 → 이사 (임원)
※ 리더 없을 시: '없음' 표시
```

**연차 / 휴가**
```
인사팀 → 임원
※ 브랜드사업본부 디자인·콘텐츠는 최성주 리더 경유
```

**품의서**
```
결재자 직접 지정 (기존 유지)
```

##### (3) 결재 양식 목록 (확장 가능)
- 일반결재
- 출장결재
- 지출결의서
- 연차 / 휴가
- 경비청구
- 품의서

## 6. 롤아웃 전략

### 6.1 단계별 오픈
```
1단계: 경영지원팀 선행 테스트 (업무보고 · 결재 · 연차)
   ↓
2단계: 사용성 · 업데이트 피드백 반영
   ↓
3단계: 키맨(리더 / 임원) 우선 오픈
   ↓
4단계: 부서별 순차 오픈
   ↓
5단계: 전사 오픈
```

### 6.2 현실 인식 및 대응
- 모든 직원이 100% 활용하기는 어려움 (어느 조직이나 공통)
- **대응 전략**:
  - 각 부서 리더 · 이사가 주도적으로 사용
  - 1차 목표는 "누가 / 어떤 프로젝트에 / 어떤 일을 하는지" 가시화
  - 자주 쓰는 기능부터 익숙해지며 확산
  - 로그인 자동 출근 등 진입 장벽 낮추기

```text
interohrigin-hr 저장소에서 HR 플랫폼 기능을 개선해줘.

[목표]
- 기능은 "경영지원팀 선행 테스트" 통과 기준으로 구현
- 기존 UX/컴포넌트 패턴을 유지하면서 필요한 화면/로직만 보완

[절대 제약]
- employees, evaluations, evaluation_items, users 테이블 ALTER 금지
- 기존 네이밍/색상/코드 패턴 유지

- 구현 후 npm run build 실행
- 결과는 변경 파일 목록 + 검증 결과 + 잔여 리스크 순서로 보고
```

### 10.2 업무보고 → 결재 연동 프롬프트
```text
업무보고 저장 직후 결재로 넘길 수 있는 "결재선 지정 → 결재 전송" 플로우를 구현해줘.

[세부 요구]
1) 업무보고 화면에 결재 전송 버튼 추가
2) 결재선 선택/수정 UI 추가
3) 결재 대기/승인/반려 상태값 연동
4) 임원 페이지에서 결재 대기 목록 즉시 조회 가능
5) 전일 임원 피드백을 다음 날 업무보고 화면에 노출

[완료 기준]
- 업무보고 1건 생성 후 결재 전송까지 단절 없이 진행
- 임원 계정에서 결재 대기 목록 확인 가능
- npm run build 통과
```

### 10.3 결재선 관리자 기능 프롬프트
```text
전자결재 결재선을 양식별/부서별로 관리자가 수정할 수 있게 구현해줘.

[세부 요구]
- 관리자/인사팀 권한에서만 결재선 수정 가능
- 지출결의: 담당자→리더→재무회계팀(2인 합의)→관리자→임원
- 일반결재: 리더→이사(리더 부재 시 '없음' 표시)
- 연차/휴가: 인사팀→임원 (일부 조직 리더 경유 예외 반영)
- 품의서: 결재자 직접 지정 유지

[완료 기준]
- 양식별 결재선 저장/조회/수정 동작 확인
- 리더 부재 케이스와 2인 합의 케이스 테스트 통과
- npm run build 통과
```

연차 관리 고도화(P-26)를 구현해줘.

[완료 기준]
- 샘플 직원 3명 시뮬레이션에서 잔여 연차 계산값 일치
- 관리자/직원 화면 모두 모바일 레이아웃 정상
- npm run build 통과
