## 2. KPI 및 목표 관리

### 현황 문제점
- 현금흐름표 수치 예측이 메신저로 수집 → 팀장들이 재고 미확인 상태로 보수적 입력
- KPI 항목이 텍스트 위주 → 숫자 목표 없음 → 추적·평가 어려움
- 행사 계획·발주 계획이 실시간 반영 안 되어 현금흐름 예측 정확도 50% 이하

### 개선 방향

#### KPI 수치화
- 담당자가 HR 시스템 내 프로젝트 계획 수립 시 **매출 목표 수치 필수 입력**
- 작년 판매 데이터 기반 목표 설정 가이드 제공
- 입력 항목:
  - 월별 목표 매출
  - 행사 계획 (날짜·채널·예상 매출)
  - 발주 계획 (발주 예정일·수량·납기)

#### 강제 입력 유도 메커니즘
- 매월 특정일까지 KPI 미입력 시 전자결재·품의서 제출 불가
- 담당자가 단계별 입력 시 자동으로 완성 → 제출 가능 상태로 변경
- 관리자가 잔소리 없이 시스템이 자동으로 유도

#### KPI 달력 연동
- 성수기(여름방학·겨울방학) 재고·현금·행사 집중 계획 달력에 표시
- 비수기 최소 재고 유지 계획 자동 경고

#### 데이터 흐름
```
담당자 KPI 입력 (매출 목표·행사 계획·발주 계획)
  → CS 판매 데이터 실적과 자동 비교
  → 재고 현황과 연동
  → 현금흐름 예측 자동 갱신
  → 대표·경영지원 대시보드에 실시간 반영
```

## 3. 전자결재 (품의서) 개선

### 현황 문제점
- 현재 네이버웍스에서 품의서 작성
- 직원들이 품의서 양식을 제대로 안 쓰고 대충 제출 ("원활한 업무 수행을 위하여...")
- 경조사·구매 요청·퀵 발송 등 각각 품의서 작성 → 담당자 검토 → 별도 결제
- 같은 데이터를 여러 곳에 이중으로 입력

### 개선 방향

#### 자동 품의서 생성
- 직원이 시스템에서 요청(경조사·구매·퀵·화환 등) 입력 시 품의서 자동 생성
- AI(Claude API) 활용: 키워드 입력 → 정해진 양식에 맞게 자동 작성
- 수정 기능 제공 (틀렸을 때 직접 수정 가능)
- 승인자가 승인만 하면 품의서 완성 → 증빙 자동 저장

#### 결제 연동 검토
- 품의서 승인 → 네이버웍스 결제 페이지 자동 연결 (또는 내부 결제)
- 중간 단계 최소화

#### 기대 효과
- 직원들이 품의서 작성 시간 절약 (현재 2분 → 목표 0분)
- 관리팀 검토 부담 감소
- 이중 입력 제거 (신청 → 품의 → 결제 → 증빙이 단일 흐름으로)

## 4. 채용 및 면접 프로세스

### 현황 문제점
- 지원 이력서 접수 → 노션 등록 → 대표 카톡으로 별도 전달
- 1차 화상면접 요약을 대표에게 별도 발송
- 대표가 면접 전 여러 채널에서 자료 취합 필요
- 담당자 연봉·평가 정보가 섞여 별도 보안 처리 필요

### 개선 방향 (HR 시스템 내 채용 모듈)

#### 대표용 면접 통합 링크
- 면접 일정 링크 하나 클릭 → 한 화면에서:
  - 이력서 + 자기소개서
  - 1차 화상면접 요약 및 평가
  - 추천 면접 질문 (1차 결과 기반 자동 생성)
- 대면 면접 후 대표가 체크:
  - 합격·불합격·보류
  - 희망연봉·제안연봉
  - 메모 입력
- 체크 완료 → 담당자에게 자동 알림

#### 지원자 관리
- 채용 공고별 지원자 목록
- 전형 단계별 상태 관리 (서류·1차·2차·최종)
- 이력서 파일 첨부 및 보관

#### 면접 스타벅스 발송 연동
- 면접 일정 확정 시 모바일 스타벅스 상품권 자동 발송 (복지몰 연계)
- 발송 내역 자동 기록 → 증빙 생성

## 5. 경조사 관리

*(복지몰 문서와 동일 내용, HR 시스템 내 모듈로 구현)*

### HR 시스템 내 경조사 메뉴 요구 사항
- 직원이 경조사 신청 시 HR 시스템에서 처리
- 모바일 청첩장·부고 링크 → 자동 파싱 (이름·날짜·장소·상주)
- 신청 → 전자결재 자동 생성 → 승인 → 발주 연동
- 데이터 누적 → 세무 증빙 자동 생성

## 6. 퀵·택배·화환 업무 요청

### 현황 문제점
- 현재 담당자가 직접 전화·사이트에서 퀵 주문 → 오션에 정리 → 담당자 확인 → 계산서 확인
- 선조치 후보고 방식 → 품의 누락 발생

### 개선 방향
- **선보고(신청) 후조치(발송) 방식으로 전환**
- 직원이 HR 시스템 내 퀵·택배 요청 메뉴에서:
  - 목적·출발지·도착지·물품 입력
  - 제이로젝스 등 업체 양식 자동 생성 (AI 작성)
  - 전자결재 자동 연결
- 승인 완료 → 업체 자동 접수 (연동 검토)
- 월별 퀵·택배 사용 현황 자동 집계 → 월 보고서 자동 생성

## 7. 디자인 레퍼런스 아카이브

### 배경
- 대표가 인스타그램 단체방에 레퍼런스 공유 → 피드백 없음, 디자이너가 찾지 못함
- 쌓인 자료 검색·분류 불가

### 요구 사항
- 업로드 시 분류 선택:
  - **브랜드:** AZH / 시몬브레드 / 베이비코너 / 공통
  - **콘텐츠 유형:** 사진 / 동영상 / 행사용 / 제품용
  - **용도:** 레퍼런스 / 제작물 / 피드백
- 필터 검색: 브랜드 + 유형 조합 검색
- 제작 결과물 피드백(OK/수정) 기록 → AI 학습 기반 방향성 축적

### 구현 위치 논의 결과
- **카카오워크 채널 + AI 봇:** 기존 채팅방 활용, 봇에게 검색 요청
- **HR 시스템 내 게시판:** 접근성·지속 사용성 우려
- **결론:** 내부 논의 후 결정 필요 (강 이사 검토)

## 9. 보안 및 접근 권한

### 현황 문제점
- 중요 데이터(직원 정보·재고·매출 등) 외부 PC·모바일에서 접근 가능
- IP 제한 어려움 (사무실 무선 환경으로 IP 고정 불가)

### 개선 방향
- 관리자 계정: 모바일 접근 제한 또는 별도 인증 추가 검토
- 권한별 메뉴 접근 제어:
  - **CS팀:** CS 상담·주문 조회·재고 현황
  - **경영지원:** 전체 메뉴 + 재무 대시보드
  - **BM/MD:** 재고 현황·행사 계획·KPI 입력
  - **일반 직원:** 복지몰·연차·경조사 신청
- 방법 검토 중 (차주용 확인 예정)

## 10. 데이터 연동 구조

### 현재
- **직원 정보**만 CS·재무·복지몰 간 공유 (단일 로그인)

### 목표 연동 구조
```
HR 플랫폼 (중심)
  ├── 직원 정보 (공통 기반)
  ├── KPI / 프로젝트 계획
  │     ↕
  ├── CS 시스템
  │     ├── 판매 데이터 (일별 누적)
  │     ├── 재고 현황
  │     └── 고객 이력
  │     ↕
  ├── 재무회계 모듈
  │     ├── 현금흐름 예측
  │     ├── 정기결제 관리
  │     └── 세금계산서 발행
  │     ↕
  └── 복지몰
        ├── 포인트 지갑
        ├── 헬스키퍼 예약
        └── 경조사 / 구매 요청
```

### 연동 시 기대 효과
- 담당자가 행사 계획 입력 → 재고·현금흐름 자동 반영
- 재고 부족 시 해당 BM에게 자동 알림
- 복지 지출 → 재무 비용 처리 자동 연결
