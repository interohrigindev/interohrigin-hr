> 기준일: 2026.05.12  
> 목적: 0507 계획서를 현재 코드 규칙, 일정 현실성, 오픈 우선순위, 비용/보안 조건까지 반영한 실행형 계획서로 재정리

## 1. 보완 목적

- 0507 계획서의 기능 목록을 실제 구현 가능한 스프린트 단위로 재편
- 현재 프로젝트 규칙과 충돌하는 항목 정리
- 오픈 우선순위를 `채용 퍼널 개선 → 채용 자동화 → AI 디자인 플랫폼` 순으로 명확화
- 각 Phase마다 `선행 조건`, `완료 기준`, `리스크`, `운영 게이트`를 추가

## 2. 보완 핵심 요약

### 2.1 즉시 수정한 계획 관점
- `자동 저장(localStorage)`는 현재 프로젝트 컨벤션과 충돌하므로 보완안에서는 제외
- 초안 복구는 `DB draft + invite token 기반 복원` 방식으로 통일
- Phase 3 디자인 플랫폼은 비용 영향이 크므로 HR 핵심 퍼널이 안정화되기 전 착수 금지
- 슬롯 선택 예약 충돌은 UI 차원이 아니라 RPC/트랜잭션 차원에서 해결하도록 명시
- 실시간 면접 콘솔은 MVP에서 `공유 메모 + 질문 추천` 중심으로 시작하고, 고급 동기화는 2차로 분리

### 2.2 일정 재산정
- 0507 문서의 `W1 (5/8~5/15)` 일정은 현재 시점(2026.05.12) 기준으로 재산정 필요
- 본 보완안에서는 2026.05.12 시작 기준으로 재배치

## 3. 공통 제약

- READ ONLY 테이블 `employees`, `evaluations`, `evaluation_items`, `users` ALTER 금지
- 한국어 UI, 날짜 `YYYY.MM.DD`, 모바일 반응형 필수
- AI 출력은 `결정`이 아닌 `제안/권장` 표현 유지
- `localStorage` / `sessionStorage` 사용 금지
- 신규 공개 링크 페이지는 읽기 전용 + 만료/비활성화 정책 포함
- 실제 구현 단계에서는 수정 후 `npm run build` 검증 필수

## 4. 우선순위 재정렬

### P0. 채용 전환율과 운영 효율에 직접 영향
- 단계형 사전 질의서
- 질의서 draft 복구
- 1차 합격 후 적성검사/일정 자동 발송
- 지원자 슬롯 선택 페이지
- 후보자 활동 타임라인

### P1. 면접 품질/판독성 향상
- 대면 면접 AI 보조 콘솔 MVP
- 보직 추천 AI 기본값 반영
- 제출 후 적성 진단 결과 요약

### P2. 비용/운영 합의 후 진행
- AI 기반 디자인 레퍼런스 플랫폼
- 비포·애프터 비교 및 어노테이션
- SMS 자동 알림 PoC
- 컬럼 단위 암호화 검토

## 5. Phase 1 — 사전 질의서 UI/UX 개선

### 목표
- 지원자가 10분 내 완료 가능한 흐름으로 재구성
- 이탈 후 복구 가능
- 관리자에게는 적성/심리 요약 결과까지 제공

### 보완 설계

#### FR-101. 단계형 UI
- `src/routes/public/survey.tsx`를 단일 페이지 나열형에서 단계형 스텝 UI로 개편
- 상단 진행률 바 + `N / M` 텍스트 + 이전/다음 버튼 제공
- 모바일에서 한 질문씩 집중하게 설계

#### FR-102. 기본 정보 분리
- Step 0에 메타데이터만 수집
- Step 1 이후에 본 질문/적성 문항 배치
- 저장 구조는 기존 `meta`, `answers` 호환 유지

#### FR-103. 적성/심리 문항 삽입
- `personality_test_items` 신규 테이블 생성
- 일반 질문 사이에 자연스럽게 분산 삽입
- 지원자에게는 별도 검사 레이블 노출 금지
- 결과는 `candidates.personality_test_result jsonb` 저장

#### FR-104. 자동 저장 / 복구
- `localStorage` 제외
- `candidates.pre_survey_draft jsonb` 컬럼에 주기 저장
- 토큰 재진입 시 draft 복원
- 제출 시 draft 삭제

#### FR-105. 제출 후 결과 요약
- 기존 AI 분석에 적성 결과를 추가 입력
- 관리자 화면에 `적성 진단 결과` 박스 노출
- 지원자에게는 미공개

### 완료 기준
- 모바일 375px에서 단계 진행 가능
- 브라우저 종료 후 재진입 시 DB draft 복구
- 10분 이내 완료 가능한 분량 확인
- 관리자 화면에서 적성 진단 결과 확인 가능

### 리스크
- 적성검사 문항 출처/라이선스
- 문항 수 과다 시 이탈률 증가

## 6. Phase 2 — 채용 프로세스 자동화

### 목표
- 1차 면접 이후 후보자 안내와 일정 조율을 자동화
- 면접관이 한 화면에서 판단 가능한 콘솔 제공

### 보완 설계

#### FR-201. 1차 합격 후 자동 발송
- `handleDecision('proceed_to_f2f')` 분기 추가
- 적성검사 메일 + 슬롯 선택 메일 동시 발송
- 상태는 `face_to_face_pending_schedule`로 전환

#### FR-202. 지원자 슬롯 선택
- `/public/interview-slot/:token` 공개 페이지 신설
- `interview_slot_options` 테이블 생성
- 예약 확정은 RPC로 처리
- 중복 예약 방지는 DB 레벨에서 강제
- 확정 후 양측 캘린더 이벤트 자동 생성

#### FR-203. 대면 면접 AI 보조 콘솔
- `/admin/recruitment/console/:candidateId` 신규
- 좌측: 지원자 요약 / 우측: 질문 추천, 답변, 점수, 메모
- MVP 범위:
  - 추천 질문 5개
  - 공유 메모
  - 면접관별 점수 입력
- 2차 범위:
  - 다중 면접관 실시간 동기화 고도화

#### FR-204. 보직 추천 AI 기본값
- 기존 종합 분석의 보직/연봉/출근일 추천값을 합격 다이얼로그 기본값으로 자동 주입
- 면접관은 수정 가능

#### FR-205. 지원자 활동 타임라인
- `candidate_activity_log` 신규
- 발송/응답/분석/상태변경 이벤트 기록
- 후보자 상세 및 공유 링크에 동일 노출

### 완료 기준
- 1차 합격 처리 시 적성검사/일정 메일 자동 발송
- 지원자가 직접 일정 선택 가능
- 일정 충돌 재현 불가
- 후보자 상세에 시간순 타임라인 표시

### 리스크
- 슬롯 동시 예약 충돌
- 캘린더 자동 생성 시 면접관 동의 정책
- 공개 링크 보안

## 7. Phase 3 — AI 기반 디자인 레퍼런스 플랫폼

### 목표
- 디자인 산출물 아카이빙, AI 추천, 비교 피드백 기능을 하나의 모듈로 제공

### 착수 조건
- Phase 1, 2 핵심 흐름 안정화
- 운영팀이 비용/용량 정책에 합의
- Storage 버킷 정책 확정

### 보완 설계

#### FR-301. 업로드/아카이빙
- `design_references` 테이블
- 이미지/링크/PDF 업로드
- 부서별 공유 권한 제어

#### FR-302. AI 추천
- Gemini multimodal 1회 자동 호출
- 재생성은 명시적 클릭으로 제한
- 카피 5종, 디자인 이슈, 수정 프롬프트 반환

#### FR-303. 비포·애프터 비교
- 좌/우 비교 슬라이더
- 코멘트 핀
- 필요 시 유사도 점수는 2차로 분리

#### FR-304. 업무 소통/보고 추적
- 스레드형 코멘트
- 변경 이력 로깅
- 해결 상태 표시

### 완료 기준
- 업로드 → AI 추천 → 결과물 비교까지 1개 흐름 완결
- 부서 권한이 적용됨
- 비용 가드가 동작함

### 리스크
- Storage 비용 급증
- 멀티모달 호출 비용
- 모바일 비교 UX 복잡도
