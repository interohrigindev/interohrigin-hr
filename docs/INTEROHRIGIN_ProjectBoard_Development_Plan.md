# 프로젝트 협업 보드 — 개발 계획서

> **기반 데이터**: ★ 프로젝트 일정 구글시트 (브랜드 사업본부 메인)
> **목표**: 구글시트를 대체하여 HR 플랫폼 내에서 프로젝트 협업 + 인사평가 연동
> **추가 비용**: $0 (기존 Supabase)
> **개발 기간**: 5~7일

---

## 1. 현재 구글시트 분석

### 1-1. 시트 구조

```
열 구조 (파이프라인):
  A: 브랜드 (AZH 등)
  B: 구분 (제품, 마케팅 등)
  C: 프로젝트명 ("(신규) 프로페셔널 플랫 헤어 스트레이트" 등)
  D: 출시/마감일
  E: 담당자 (복수: "백지영 / 이진희")
  G: 진행 메모 (자유 서술, 매일 아침 9:30까지 업데이트)
  H~U: 7단계 파이프라인 (각각 진행상태 + 마감일)

7단계 파이프라인:
  ① 시장조사 → ② 제형 → ③ 패키지 → ④ 판매가 → ⑤ 상세페이지 → ⑥ 촬영 → ⑦ 마케팅

진행상태 값:
  완료 / 진행중 / 시작전 / 홀딩

현재 프로젝트 수: 약 17개+ (AZH 브랜드 중심)
담당자: 백지영, 정유리, 이진희, 김푸른, 김윤정 등
```

### 1-2. 현재 문제점

```
❌ 매일 구글시트를 직접 열어서 수동 업데이트 (9:30 루틴)
❌ 진행 메모(G열)가 자유 서술이라 내용이 길고 구조화 안 됨
❌ 타 부서(디자인, 영업, 경영지원)는 읽기만 가능하고 협업 연결 안 됨
❌ 파이프라인 단계별 지연을 한눈에 파악 어려움
❌ 업무 데이터가 인사평가와 연결 안 됨
❌ 알림/리마인드 없음 — 마감일 넘겨도 모름
❌ 프로젝트 히스토리가 없음 — 과거 변경 이력 추적 불가
```

## 2. 솔루션: 프로젝트 협업 보드

### 2-1. 핵심 기능

```
┌─────────────────────────────────────────────────────┐
│  프로젝트 협업 보드 — 구글시트를 대체하는 것이 아니라  │
│  구글시트의 장점을 살리되, 더 강력하게 만드는 구조     │
│                                                     │
│  ① 파이프라인 뷰 — 7단계 진행 상태를 한눈에           │
│  ② 칸반 보드 — 드래그&드롭으로 상태 변경              │
│  ③ 권한 분리 — 브랜드팀 편집 / 타부서 읽기+코멘트     │
│  ④ AI 자동화 — 마감 알림, 진행 메모 구조화, 요약      │
│  ⑤ 협업 연결 — 타부서 요청을 보드에서 직접 처리        │
│  ⑥ 인사평가 연동 — 프로젝트 기여도 자동 산출           │
│  ⑦ 실시간 동기화 — Supabase Realtime으로 즉시 반영    │
└─────────────────────────────────────────────────────┘
```

### 2-2. 뷰 모드 3종

```
뷰 1: 테이블 뷰 (구글시트 익숙한 사용자용)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  기존 구글시트와 거의 동일한 레이아웃.
  브랜드 | 프로젝트명 | 출시일 | 담당자 | 시장조사 | 제형 | 패키지 | ...
  
  + 인라인 편집 (셀 클릭 → 바로 수정)
  + 상태는 드롭다운 (완료/진행중/시작전/홀딩)
  + 마감일 지난 셀은 자동 빨간색
  + 정렬/필터: 브랜드별, 담당자별, 상태별, 마감순

뷰 2: 칸반 보드 (시각적 관리)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐
  │시장조사 │→│ 제형   │→│패키지  │→│ 판매가 │→│상세페이지│→│ 촬영  │→│마케팅  │
  │        │ │        │ │        │ │        │ │         │ │       │ │       │
  │ 탈모토닉│ │스팀팩  │ │실키오일│ │머리끈  │ │헤어스틱 │ │       │ │       │
  │ 괄사   │ │사우나팩│ │데일리  │ │        │ │         │ │       │ │       │
  └────────┘ └────────┘ └────────┘ └────────┘ └────────┘ └────────┘ └────────┘
  
  각 카드를 드래그&드롭으로 다음 단계로 이동.
  카드 색상: 정상=초록, 지연=주황, 초과=빨강, 홀딩=회색.

뷰 3: 타임라인 뷰 (간트 차트)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  프로젝트별 가로 막대로 일정 시각화.
  2월 ─────── 3월 ─────── 4월 ─────── 5월
  ██ 스트레이트  ████████████████████
      ████ 스팀팩  ████████████████████████
         ██████ 실키오일  ████████
  
  겹치는 일정, 병목 구간을 한눈에 파악.
```

### 2-3. 권한 체계

```
역할별 권한:

  브랜드 사업본부 (BM팀):
    ✅ 프로젝트 생성/삭제
    ✅ 모든 필드 편집 (상태, 메모, 마감일, 담당자)
    ✅ 파이프라인 단계 변경
    ✅ 파일 첨부 (시안, 견적서 등)
    ✅ 타부서 요청 생성

  디자인팀:
    ✅ 읽기 전체
    ✅ "패키지" + "상세페이지" 단계만 편집 가능
    ✅ 코멘트 작성
    ✅ 파일 첨부 (디자인 시안)
    ❌ 프로젝트 생성/삭제 불가

  영업팀:
    ✅ 읽기 전체
    ✅ "판매가" + "마케팅" 단계만 편집 가능
    ✅ 코멘트 작성
    ❌ 다른 단계 편집 불가

  경영지원팀:
    ✅ 읽기 전체
    ✅ 코멘트 작성
    ❌ 편집 불가

  임원:
    ✅ 전체 읽기/편집
    ✅ 프로젝트 승인/반려
    ✅ 우선순위 변경

  권한 설정은 관리자(설정 페이지)에서 부서별 + 단계별로 세밀 조정 가능.
```

## 3. 소모적 반복 업무 해결 방안

### 3-1. AI 아침 자동 요약 (9:30 업데이트 대체)

```
현재: 매일 아침 BM이 구글시트를 열어서 수동으로 진행 상황 서술

변경: AI가 전날 변경사항을 자동 요약하여 아침에 제공

  ┌──────────────────────────────────────────────────┐
  │  🌅 오늘의 프로젝트 현황 (2026.03.19 09:00 자동)  │
  │                                                  │
  │  📊 전체: 17개 프로젝트 | 정상 12 | 지연 3 | 홀딩 2│
  │                                                  │
  │  ⚠ 주의 필요:                                    │
  │  · 스팀헤어팩: 패키지 시안 오늘 마감 (무조건 끝내기) │
  │  · 실키오일: 입고 D-1, 상세페이지 작업 중           │
  │  · 머리끈: 판매가 높아 진행 여부 결정 필요          │
  │                                                  │
  │  ✅ 어제 완료:                                    │
  │  · 스트레이트: 데이터 목업 수정 전달 완료            │
  │  · 사우나팩: 타겟제품 도착                         │
  │                                                  │
  │  📋 오늘 해야 할 일:                              │
  │  · 백지영: PL보험 견적 비교 보고                    │
  │  · 정유리: 스팀팩 패키지 시안 마감                   │
  │  · 김윤정: 두피쿨링팩 제형 샘플 요청                │
  │                                                  │
  │  [전체 보드 보기] [내 할 일 보기]                   │
  └──────────────────────────────────────────────────┘

→ 매일 아침 자동 생성, 메신저/이메일로 발송.
→ BM은 시트 직접 편집 대신 "할 일 완료" 버튼만 클릭.
→ 상세 메모가 필요하면 프로젝트 카드에 코멘트로 남김.
```

### 3-2. 타부서 요청 자동화

```
현재: "디자인팀, 이 시안 좀 해주세요" → 카톡/슬랙으로 전달 → 까먹음

변경: 프로젝트 카드에서 바로 요청

  프로젝트 "스팀헤어팩" → 패키지 단계 클릭:
  ┌──────────────────────────────────────┐
  │  📦 패키지 — 스팀헤어팩 리뉴얼       │
  │  상태: 진행중 | 마감: 3/19           │
  │                                      │
  │  [📎 시안 첨부] [💬 코멘트]           │
  │                                      │
  │  [🔔 디자인팀에 요청]                 │
  │  → "패키지 시안 오늘까지 마감입니다.   │
  │     단상자 문안 확인 부탁드립니다."    │
  │  → 디자인팀에 알림 + 메신저 발송       │
  │  → 완료 시 자동으로 상태 변경          │
  └──────────────────────────────────────┘

타부서 요청이 프로젝트 카드에 기록으로 남아서:
  누가 언제 요청했는지
  언제 완료했는지
  몇 번 리마인드 됐는지
  → 부서간 협업 이력이 자동 축적
```

### 3-3. 마감일 자동 알림 + 연쇄 알림

```
현재: 마감일 넘겨도 아무 알림 없음

변경: 단계별 마감일 기반 자동 알림

  D-3: "패키지 마감 3일 전입니다" (담당자에게)
  D-1: "내일이 마감입니다. 현재 상태를 업데이트해주세요" (담당자 + 리더)
  D-0: "오늘이 마감일입니다" (팀 전체)
  D+1: "⚠ 마감 초과! 사유를 입력해주세요" (담당자 + 임원)

  연쇄 알림:
  패키지가 지연되면 → 다음 단계(판매가, 상세페이지)의 일정도 자동 재계산
  → "패키지가 3일 지연되어 상세페이지 마감이 3/23→3/26으로 조정됩니다"
  → 관련 담당자 전원에게 알림
```

### 3-4. 진행 메모 구조화

```
현재: G열에 자유 서술 (길고 읽기 어려움)

변경: 구조화된 업데이트 로그

  프로젝트 카드에 "업데이트 추가" 클릭:
  ┌──────────────────────────────────────┐
  │  업데이트 추가 — 스팀헤어팩            │
  │                                      │
  │  관련 단계: [패키지 ▼]               │
  │  상태 변경: [진행중 → 완료 ▼]        │
  │                                      │
  │  내용: [패키지 시안 최종 확인 완료.    │
  │         권래은님 전달 완료.]           │
  │                                      │
  │  첨부: [📎 시안_최종.pdf]             │
  │                                      │
  │  [등록]                               │
  └──────────────────────────────────────┘

→ 타임라인으로 쌓여서 프로젝트 히스토리가 됨
→ 누가 언제 무엇을 했는지 추적 가능
→ AI가 요약하여 "이번 주 진행 사항" 자동 생성
```

### 3-5. 반복 프로젝트 템플릿

```
현재: 신제품 프로젝트 만들 때마다 같은 단계를 수동 설정

변경: 템플릿으로 한 번에 생성

  "새 프로젝트" 클릭 → 템플릿 선택:
  ○ 신제품 출시 (7단계: 시장조사→마케팅, 평균 4개월)
  ○ 리뉴얼 (5단계: 제형→마케팅, 평균 2개월)
  ○ 단순 용기 변경 (3단계, 평균 1개월)
  ○ 커스텀

  선택 시 → 각 단계의 예상 마감일이 자동 계산
  → 과거 유사 프로젝트의 실제 소요일 참고
  → 담당자 추천 (현재 업무량 기반)
```

## 4. Supabase 테이블 설계

```sql
-- 프로젝트 보드 (구글시트 대체)
CREATE TABLE project_boards (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  
  -- 프로젝트 기본 정보
  brand text NOT NULL,                  -- 'AZH', '드엘리사' 등
  category text DEFAULT '제품',          -- '제품' | '마케팅' | '이벤트'
  project_name text NOT NULL,
  launch_date date,                     -- 출시/마감일
  status text DEFAULT 'active',         -- 'active' | 'holding' | 'completed' | 'cancelled'
  priority integer DEFAULT 5,           -- 1(최고)~10
  
  -- 담당자
  assignee_ids uuid[] NOT NULL,         -- 복수 담당자
  department text DEFAULT '브랜드사업본부',
  
  -- 템플릿
  template_type text,                   -- 'new_product' | 'renewal' | 'repackage' | 'custom'
  
  created_by uuid,
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);

-- 파이프라인 단계 (7단계)
CREATE TABLE pipeline_stages (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  project_id uuid REFERENCES project_boards(id) ON DELETE CASCADE,
  
  stage_name text NOT NULL,             -- '시장조사' | '제형' | '패키지' | '판매가' | '상세페이지' | '촬영' | '마케팅'
  stage_order integer NOT NULL,         -- 1~7
  
  status text DEFAULT '시작전',          -- '완료' | '진행중' | '시작전' | '홀딩'
  deadline date,
  completed_at timestamptz,
  
  -- 부서별 편집 권한
  editable_departments text[] DEFAULT '{"브랜드사업본부"}',
  
  -- 담당자 (단계별로 다를 수 있음)
  stage_assignee_ids uuid[],
  
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);

-- 프로젝트 업데이트 로그 (진행 메모 대체)
CREATE TABLE project_updates (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  project_id uuid REFERENCES project_boards(id) ON DELETE CASCADE,
  stage_id uuid REFERENCES pipeline_stages(id),  -- 특정 단계 관련 (선택)
  
  author_id uuid NOT NULL,
  content text NOT NULL,
  
  -- 상태 변경 기록
  status_changed_from text,             -- 이전 상태
  status_changed_to text,               -- 새 상태
  
  -- 첨부파일
  attachments jsonb DEFAULT '[]',       -- [{url, name, size, type}]
  
  -- 타부서 요청
  is_cross_dept_request boolean DEFAULT false,
  requested_department text,            -- 요청 대상 부서
  request_status text,                  -- 'pending' | 'accepted' | 'completed'
  request_completed_at timestamptz,
  
  created_at timestamptz DEFAULT now()
);

-- 프로젝트 보드 권한 설정
CREATE TABLE board_permissions (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  department text NOT NULL,
  
  -- 전체 권한
  can_create_project boolean DEFAULT false,
  can_delete_project boolean DEFAULT false,
  can_edit_all_stages boolean DEFAULT false,
  can_comment boolean DEFAULT true,
  can_view boolean DEFAULT true,
  
  -- 단계별 편집 권한
  editable_stages text[] DEFAULT '{}',  -- ['패키지', '상세페이지'] 등
  
  updated_at timestamptz DEFAULT now()

```

```

  ┌──────────────────────┐
  │ AZH | 제품            │
  │ 스팀헤어팩 리뉴얼      │
  │ 📅 3/19 | ⚠ D-0      │
  │ 👤 정유리, 이진희      │
  │ 💬 3 | 📎 2           │
  └──────────────────────┘

카드 색상: 마감 여유=초록 테두리, D-3이내=주황, 초과=빨강.
드래그 시 권한 체크 (편집 불가 단계로는 이동 불가).

## STEP 3: 타임라인 뷰 (간트 차트)

가로축: 날짜 (주 단위)
세로축: 프로젝트 리스트

각 프로젝트의 7단계가 가로 막대로 표시.
  시장조사 ███ 제형 █████ 패키지 ███ ...

색상: 완료=초록, 진행중=파랑, 시작전=연회색, 홀딩=주황.
오늘 날짜에 빨간 세로선.

## STEP 4: Supabase Realtime 동기화

pipeline_stages 테이블에 Realtime 구독:
  다른 사용자가 상태를 변경하면
  → 테이블/칸반/타임라인 뷰 모두 즉시 반영
  → 변경한 사용자 이름 + 변경 내용 토스트 알림

## 주의사항
- 칸반 드래그&드롭: react-beautiful-dnd 또는 @dnd-kit 사용
- 타임라인: recharts 또는 기존 차트 라이브러리
- 모바일에서 칸반은 가로 스크롤, 타임라인은 세로 전환
```

### P-BOARD-03 — 프로젝트 상세 + 업데이트 로그 + 타부서 요청

```
개별 프로젝트의 상세 화면과 협업 기능입니다.

## STEP 1: 프로젝트 상세 페이지

/admin/projects/:id

상단: 프로젝트 기본 정보 (편집 가능)
  브랜드, 이름, 출시일, 담당자, 상태, 우선순위

중단: 파이프라인 진행 상황 (가로 스텝 바)
  ✅시장조사 → ✅제형 → 🔄패키지 → ⏳판매가 → ⏳상세 → ⏳촬영 → ⏳마케팅

하단 탭: [📝 업데이트] [📎 파일] [💬 코멘트] [🔔 요청] [📊 통계]

## STEP 2: 업데이트 로그 (진행 메모 구조화)

기존 구글시트 G열의 자유 서술을 구조화:

  "업데이트 추가" 클릭:
    관련 단계: [드롭다운]
    상태 변경: [선택적]
    내용: [텍스트]
    첨부: [파일]
  
  → 타임라인으로 표시 (최신순)
  → 각 업데이트에 코멘트 가능
  → 상태 변경 시 자동 업데이트 생성: "패키지: 진행중 → 완료 (정유리)"

## STEP 3: 타부서 요청 시스템

"🔔 요청" 탭에서:
  [+ 새 요청] 클릭:
    대상 부서: [디자인팀 ▼]
    관련 단계: [패키지 ▼]
    요청 내용: [단상자 시안 검토 부탁드립니다]
    마감: [3/22]
  
  → 대상 부서에 알림 (메신저 + 보드 배지)
  → 대상 부서가 "수락" → "완료" 처리
  → 완료 시 요청자에게 알림
  → 미완료 시 자동 리마인드

요청 상태: 요청중 → 수락 → 완료 (또는 반려+사유)

## STEP 4: 파일 관리

프로젝트별 파일 첨부:
  시안, 견적서, 제형 데이터, 사진 등
  단계별로 구분하여 정리
  Supabase Storage project-files 버킷

## STEP 5: 프로젝트 통계

📊 통계 탭:
  - 평균 단계별 소요일 vs 이 프로젝트 실제 소요일
  - 지연된 단계 하이라이트
  - 담당자별 기여도 (업데이트 수, 완료 단계 수)
  → 인사평가 work_metrics에 연동 가능
```

### P-BOARD-04 — AI 자동화 (아침 요약 + 알림 + 템플릿)

```

## STEP 1: 아침 자동 요약 (9:30 대체)

매일 오전 9:00에 AI가 자동 생성:
  - 전일 변경 사항 요약
  - 오늘 마감 업무 리스트
  - 지연 프로젝트 경고
  - 담당자별 오늘 할 일

배포: 메신저 "전체 공지" 채팅방 + 이메일 (선택)

AI 프롬프트:
  "프로젝트 관리자로서 오늘의 프로젝트 현황을 간결하게 요약하세요.
   전체 프로젝트: {project_boards 데이터}
   어제 변경사항: {project_updates에서 어제 날짜 데이터}
   지연 항목: {deadline < today인 pipeline_stages}
   
   형식: 주의 필요(⚠) → 어제 완료(✅) → 오늘 할 일(📋)"

## STEP 2: 마감일 자동 알림

Supabase Edge Function (스케줄 또는 cron):
  매일 오전 확인 → D-3, D-1, D-0, D+1 알림 발송
  
  연쇄 일정 재계산:
  한 단계가 지연되면 → 이후 단계의 시작일/마감일 자동 조정
  → 관련 담당자에게 "일정이 변경되었습니다" 알림

## STEP 3: 프로젝트 템플릿 + AI 일정 추천

"새 프로젝트" 생성 시:
  1. 템플릿 선택 (신제품/리뉴얼/리패키지)
  2. 출시 목표일 입력
  3. AI가 역산하여 각 단계 마감일 자동 설정
     (과거 유사 프로젝트의 실제 소요일 참고)
  4. 담당자 추천 (현재 업무량 기반)
  5. 미리보기 → 수정 → 확정

## STEP 4: 주간 리포트 자동 생성

매주 금요일 자동 생성:
  "이번 주 프로젝트 진행 요약"
  - 완료된 단계: 8개
  - 지연된 단계: 2개 (사유)
  - 다음 주 주요 마감: 5건
  - 부서별 요청 처리 현황: 디자인팀 3/4건 완료

→ 임원에게 메신저/이메일로 발송
```
