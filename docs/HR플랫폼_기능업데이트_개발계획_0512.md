## 1. 일일 보고서 개편

### 배경
- 블록 구조 복잡 → 직원 사용성 저하
- 프로젝트 작업 업데이트 + 일일 보고서 작성 이중 작업 인식
- 먼데이닷컴 병행 사용으로 혼란 → 일원화 필요
- **목표:** 프로젝트 작업 업데이트 = 일일 보고서 자동 완성

### 블록 구성 변경

#### 유지할 블록 (3개)
| 블록명 | 내용 |
|--------|------|
| 작업 현황 | 오늘 업데이트한 작업 (프로젝트에서 자동 연동) |
| 진행 중 작업 | 현재 진행 중인 파이프라인 단계 작업 |
| 오늘의 총평 | 결재자 코멘트 + 작성자 한줄 총평 |

#### 숨김 처리할 블록
- 미완료 작업 목록
- 내일 계획
- 만족도

> 숨김 처리 후 운영하다가 필요 시 다시 노출 가능하도록 설정 유지

### 업데이트 항목
| 항목 | 내용 | 우선순위 |
|------|------|----------|
| UI 간소화 | 핵심 3개 블록만 노출, 나머지 숨김 | 🔴 높음 |
| 작업 자동 연동 | 프로젝트 담당 작업이 일일 보고서에 자동 반영 | 🔴 높음 |
| 코멘트 기능 | 결재자 승인 시 코멘트 유지 | 🔴 높음 |
| 자유도 향상 | 텍스트/표/이미지 자유 입력 게시판형 포맷 | 🟡 중간 |
| 저장 버튼 조건부 활성화 | 당일 마감 미완료 작업 있으면 저장 비활성 | 🟡 중간 |
| 메모 영역 확장 | 메모 입력 칸 높이 확대 | 🟢 낮음 |

### 세부 로직: 저장 버튼 조건부 활성화
```
트리거: 오늘 마감 기한으로 설정된 작업 중 '완료' 처리되지 않은 항목 존재 시
동작:   저장 버튼 비활성화
안내:   "오늘 마감 작업이 완료되지 않았습니다. 작업을 완료하거나 마감일을 조정해주세요."
기준 시각: 18:00
```

## 2. 프로젝트 관리 개선

### 배경
- 프로젝트 생성/담당자 지정/파이프라인 관리는 프로젝트 생성자(담당자)가 수행
- 일반 참여자는 자기 작업 업데이트만 → 일일 보고서까지 자동 연결
- 프로젝트 모르는 직원도 일일 보고서 작성 가능하도록 진입 장벽 낮춤

### 업데이트 항목
| 항목 | 내용 | 우선순위 |
|------|------|----------|
| 파이프라인 단계별 담당자 복수 지정 | 각 단계마다 담당자 여러 명 | 🔴 높음 |
| 담당자 기반 작업 자동 노출 | 담당 단계 작업이 일일 보고서 자동 연동 | 🔴 높음 |
| 대시보드 권한 분리 | 참여자=참고용 / 수정=담당자·생성자 | 🟡 중간 |
| 작업 상세 뷰 개선 | 첨부 이미지·내용이 리스트에서 바로 노출 | 🟡 중간 |
| 참고용 UI 표시 | 프로젝트 보드·작업관리가 참고용임을 UI에 명확히 | 🟢 낮음 |

### 권한 구조 정리
| 역할 | 가능한 작업 |
|------|-------------|
| 프로젝트 생성자 | 프로젝트 생성, 파이프라인 구성, 담당자·참여자 지정, 단계 변경 |
| 파이프라인 담당자 | 해당 단계 상태 변경, 작업 추가·완료 처리 |
| 참여자 | 자기 작업 업데이트·완료 처리, 대시보드 참고 |

## 5. 채용관리 개선

### 배경
- 현행 7단계 고정 파이프라인 → 사전지원서 단계의 유연한 처리 요청
- 이력서 접수 → AI 분석 → 면접 일정 확정 → 사전지원서 수집 순서로 변경
- 내부 스펙티브 결과 기반 채용 판단용 사전지원서로 콘텐츠 개편

### 채용 단계 재구성

#### 현행 (7단계)
```
서류 접수 → AI 이력서 분석 → 사전지원서 → 면접 일정 확정 → 대면 면접 → 처우 협의 → 최종 합격
```

#### 변경안
```
서류 접수 → AI 이력서 분석 → 면접 일정 확정 → 대면 면접 → 처우 협의 → 최종 합격
                                    ↕ (언제든 별도 발송 가능)
                               사전지원서 요청
```

### 업데이트 항목
| 항목 | 내용 | 우선순위 |
|------|------|----------|
| 채용 단계 재구성 | 사전지원서를 고정 단계에서 제거, 별도 요청 방식 | 🔴 높음 |
| 사전지원서 콘텐츠 개편 | 스펙티브 기반 20~30문항 내부 버전 제작 | 🔴 높음 |
| 단계 Skip 기능 | 담당자 권한으로 특정 단계 건너뛰기 | 🟡 중간 |
| 샘플 데이터 수집 | 내부 직원 스펙티브 결과 3~5명 | 🟡 중간 |

## 6. 인수인계 기능

### 배경
- 퇴사 예정자 설정 시 인수인계서 작성 메뉴 자동 노출
- Claude API 활용 — 프로젝트/업무 보고 데이터 기반 초안 자동 생성

### 노출 조건 설계
```

동작:   해당 직원에게만 "나의 인수인계" 메뉴 노출
해제:   퇴사 예정자 플래그 해제 시 메뉴 자동 숨김
참고:   OJT 메뉴와 동일한 조건부 노출 방식 적용
```

### 업데이트 항목
| 항목 | 내용 | 우선순위 |
|------|------|----------|
| 조건부 메뉴 노출 | 퇴사 예정자 플래그 기반 노출/숨김 | 🔴 높음 |
| AI 인수인계서 초안 생성 | 프로젝트 + 일일 보고서 → Claude API 종합 초안 | 🔴 높음 |
| 인수인계서 편집 에디터 | 초안 확인 후 직접 추가·수정 | 🟡 중간 |
| 플래그 해제 시 자동 숨김 | 퇴사 예정자 해제 → 메뉴 즉시 비노출 | 🟡 중간 |

## 7. 정규직 평가 — 긴급 수정 (이번 PR)

### 사용자 보고 이슈 (스크린샷 기반)

| # | 이슈 | 우선순위 |
|---|------|----------|
| 7-1 | 점수 입력이 select 드롭다운 → 가로형 1~10 단계 선택으로 변경 | 🔴 높음 |
| 7-2 | "평가 확정" 버튼 계속 노출 → 누르면 `evaluator_scores` RLS 에러 | 🔴 높음 |
| 7-3 | 강제묵이 차주용 평가 완료했는데 목록에서 "내 평가 필요" 로 표시 | 🔴 높음 |
| 7-4 | 평가 단계 순서 재정의 (아래 7-5) | 🔴 높음 |
| 7-5 | 평가 흐름 정의:<br>• **팀 리더 있음**: 자기평가 → 팀 리더 → 부서 임원 → 타부서 임원 → 대표<br>• **팀 리더 없음**: 자기평가 → 부서 임원 → 타부서 임원 → 대표<br>• 단, 임원 단계는 임원 간 순서 무관 (누구나 먼저 가능) | 🔴 높음 |

### 작업 항목

**A. UI**
- 점수 입력을 가로형 1~10 토글 버튼 그룹으로 (`<button>×10` 가로 배열, 선택 시 brand 색상)
- 평가 확정 버튼 노출 조건: 모든 점수 + 종합 코멘트가 채워졌고, 아직 본인 단계 미완료일 때만

**B. RLS 점검**
- `evaluator_scores` insert 정책 검토 — `evaluator_id = auth.uid()` 일치 보장 + `evaluation_targets` 단계와 일치 검증
- target.evaluator_id 미할당으로 인한 RLS 거부 가능성 → 자동 배정 후 insert

**C. 평가자 대기 리스트 쿼리 (7-3)**
- 강제묵 케이스: 평가 row 는 들어갔는데 evaluation_targets.status 가 단계 전환 안 됨 → 목록 쿼리가 `status='self_done' AND leader_reviewed_at IS NULL` 기준으로 가져옴 → 이미 완료된 사람도 다시 표시
- 해결: 평가 row 와 status 동시 일관성 보장, 또는 쿼리에서 본인이 이미 평가한 target 제외 (`NOT EXISTS evaluator_scores WHERE evaluator_id=me AND target_id=t.id`)

**D. 단계 전환 로직 (7-5)**
- DB: `evaluation_targets.status` 의 enum 확장 또는 새 컬럼 `stage` 도입
  - `pending` → `self_done` → (`leader_done` if 팀리더 있음 else skip) → `exec_done` (임원 다수 중 1명이 처리) → `ceo_done` → `completed`
- `evaluation_workflow` 재구현: 대상자의 부서 → 팀 리더 존재 여부 판정 → 다음 평가자 후보 결정
- 임원 단계는 동시 가능 (병렬 처리). 한 임원이 평가 시작하면 lock 또는 다른 임원에게 알림만 (cooperative)

## 8. 월간 업무 점검 개편

- 팀원 월간 점검 목록이 평면적인 시간순 정렬
- 임원/대표 화면에서 전체 직원 점검을 보면 부서별·인원별 그룹화 부재 → 탐색 어려움

### 업데이트 항목
| 항목 | 내용 | 우선순위 |
|------|------|----------|
| 임원/대표 화면 부서별 그룹핑 | 사이드바 또는 상단 탭으로 부서 선택 | 🔴 높음 |
| 직원별 펼치기/접기 | 각 직원 카드 펼치면 월별 점검 누적 표시 | 🔴 높음 |
| 검색·필터 | 직원명/상태(임원 검토/대표 검토/제출됨) 필터 | 🟡 중간 |

### 권장 구조 (추가)
```
[부서 탭]  경영지원  |  마케팅영업본부  |  브랜드사업본부
   ↓
[직원 카드 리스트]  ▸ 차주용  (개발)
                     ㄴ 4월 [기타] 임원 검토   [피드백 작성]
                     ㄴ 5월 [기타] 제출됨    [피드백 작성]
                  ▸ 이민지  (인사총무매니저)
                     ㄴ 4월 [기타] 대표 검토
```
