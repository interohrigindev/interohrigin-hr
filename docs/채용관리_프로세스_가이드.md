# 인터오리진 HR — 채용관리 프로세스 가이드

> 최종 업데이트: 2026.03.31
> 버전: 1.0

---

## 전체 흐름 (13단계)

```
지원서 접수 → AI 이력서 분석 → 사전질의서 발송 → 질의서 응답
→ 1차 화상면접 예약 → 1차 면접 진행 → 면접 파일 업로드/분석
→ 2차 대면면접 예약 → 2차 면접 진행 → 면접 파일 업로드/분석
→ 최종 종합 AI 분석 → 합격/불합격 결정 → 합격 조건 응답
```

### 상태 흐름도

```
applied → resume_reviewed → survey_sent → survey_done
  → interview_scheduled → video_done
  → face_to_face_scheduled → face_to_face_done
  → processing → analyzed → decided
  → hired 또는 rejected
```

## 단계별 상세

### 1단계: 지원서 접수 (applied)

| 항목 | 내용 |
|------|------|
| 경로 | `/careers` → `/apply/:postingId` |
| 주체 | 지원자 (외부, 로그인 불필요) |
| 입력 | 이름, 이메일, 전화번호, 이력서(PDF/이미지), 자기소개서(파일 또는 텍스트) |
| 파일 제한 | PDF, PNG, JPG만 허용 (docx 불가) |
| 저장 위치 | `candidates` 테이블 + `resumes` Storage 버킷 (private) |
| 자동 처리 | invite_token 자동 생성, 상태 `applied` |

---

### 2단계: AI 이력서 분석 (applied → resume_reviewed)

| 항목 | 내용 |
|------|------|
| 경로 | 지원자 상세 `/admin/recruitment/candidates/:id` |
| 주체 | HR 관리자 |
| 동작 | "AI 분석" 버튼 클릭 |
| AI 엔진 | Gemini 2.5 Flash |
| 분석 내용 | 직무 적합도, 조직 적합도, 강점, 약점, 주의사항, 추천 |
| 결과 저장 | `resume_analysis` 테이블 |
| 추천값 | PROCEED / REVIEW / REJECT |
| 상태 변경 | → `resume_reviewed` |

**분석 결과 항목:**
- `position_fit`: 직무 적합도 (0-100점)
- `organization_fit`: 조직 적합도 (0-100점)
- `strengths`: 강점 배열
- `weaknesses`: 약점 배열
- `red_flags`: 주의사항 배열
- `recommendation`: AI 추천 (PROCEED/REVIEW/REJECT)
- `ai_summary`: 종합 요약

---

### 3단계: 사전질의서 발송 (resume_reviewed → survey_sent)

| 항목 | 내용 |
|------|------|
| 주체 | HR 관리자 |
| 동작 | "OK — 사전 질의서 발송" 버튼 클릭 |
| 이메일 | `surveyInviteEmail()` 발송 → 지원자 이메일 |
| 내용 | 질의서 작성 링크 포함 (`/survey/:invite_token`) |
| 질의서 템플릿 | 공고별 설정 (경영관리, 브랜드사업, 마케팅영업 등) |
| 상태 변경 | → `survey_sent` |

**또는 불합격:**
- "불합격 처리" 버튼 → 상태 `rejected` + 불합격 이메일 발송

---

### 4단계: 사전질의서 응답 (survey_sent → survey_done)

| 항목 | 내용 |
|------|------|
| 경로 | `/survey/:invite_token` (외부 페이지) |
| 주체 | 지원자 |
| 동작 | 질의서 작성 후 제출 |
| 저장 | `candidates.pre_survey_data` (JSONB) |
| 상태 변경 | → `survey_done` (자동) |

**관리자 추가 동작:**
- 질의서 응답 포함 재분석 가능 ("사전질의서 포함 재분석" 버튼)
- 재발송 가능 ("사전질의서 재발송" → 응답 초기화 + 재발송)

---

### 5단계: 1차 화상면접 예약 (survey_done → interview_scheduled)

| 항목 | 내용 |
|------|------|
| 경로 | `/admin/recruitment/schedules` |
| 주체 | HR 관리자 |
| 방법 | 수동 일정 추가 또는 AI 자동 배정 |
| 자동 생성 | Google Meet 링크 자동 생성 (Calendar API) |
| 이메일 | `interviewInviteEmail()` — 일시, Meet 링크, 안내사항 포함 |
| 상태 변경 | → `interview_scheduled` |

**AI 자동 배정:**
- 날짜/시간대/면접 시간/쉬는 시간 설정
- AI가 우선순위 판단하여 겹치지 않게 자동 배정
- `survey_done` 상태 지원자 자동 인식

---

### 6단계: 1차 화상면접 진행 + 완료 (interview_scheduled → video_done)

| 항목 | 내용 |
|------|------|
| 주체 | 면접관 |
| 동작 | Google Meet에서 면접 진행 |
| 완료 처리 | 면접일정 관리에서 "완료" 처리 |
| 상태 변경 | → `video_done` |

---

### 7단계: 면접 파일 업로드 + AI 분석

| 항목 | 내용 |
|------|------|
| 경로 | 지원자 상세 > 면접 분석 섹션 |
| 주체 | HR 관리자 |
| 파일 | 녹화 영상 (mp4, webm 등) 또는 Google Drive에서 자동 가져오기 |
| 저장 | `interview-recordings` Storage 버킷 (private) |
| AI 분석 | "AI 내용 분석" 버튼 → Gemini API |
| 분석 내용 | 전사(STT), 핵심답변, 소통점수, 전문성점수, 태도점수, 종합점수 |
| 확인 완료 | "확인 완료" 클릭 → 원본 파일 14일 보관 후 자동 삭제, 분석 텍스트 영구 보관 |

**바로보기/다운로드:**
- Signed URL(1시간) 사용 (private 버킷)
- 확인 완료 후에도 14일간 접근 가능

**분석 결과 항목:**
- `transcription`: 전사 텍스트
- `key_answers`: 핵심 Q&A + 평가
- `communication_score`: 소통 능력 (0-100)
- `expertise_score`: 전문성 (0-100)
- `attitude_score`: 태도 (0-100)
- `overall_score`: 종합 점수 (0-100)
- `strengths`: 강점 배열
- `concerns`: 우려사항 배열

---

### 8단계: 2차 대면면접 예약 (video_done → face_to_face_scheduled)

| 항목 | 내용 |
|------|------|
| 경로 | `/admin/recruitment/schedules` |
| 주체 | HR 관리자 |
| 방법 | 수동 일정 추가 또는 AI 자동 배정 |
| Meet 링크 | 생성 안 함 (현장 면접) |
| 이메일 | `interviewInviteEmail()` — 일시, 장소 안내 포함 |
| 상태 변경 | → `face_to_face_scheduled` |

**AI 자동 배정:**
- `video_done` 상태 지원자 자동 인식 → `face_to_face`로 배정
- Meet 링크 생성하지 않음

---

### 9단계: 2차 대면면접 진행 + 완료 (face_to_face_scheduled → face_to_face_done)

| 항목 | 내용 |
|------|------|
| 주체 | 면접관 |
| 동작 | 현장 대면면접 진행 |
| 녹음 업로드 | 녹음 파일(mp3, wav 등) 업로드 가능 |
| AI 분석 | 7단계와 동일 (오디오 분석) |
| 완료 처리 | 면접일정 관리에서 "완료" 처리 |
| 상태 변경 | → `face_to_face_done` |

---

### 10단계: 최종 종합 AI 분석 (face_to_face_done → analyzed)

| 항목 | 내용 |
|------|------|
| 경로 | 지원자 상세 > 최종 종합 분석 섹션 |
| 주체 | HR 관리자 |
| 동작 | "최종 종합 분석" 버튼 클릭 |
| AI 엔진 | Gemini 2.5 Flash |
| 입력 데이터 | 이력서 분석 + 사전질의서 + 1차 면접 분석 + 2차 면접 분석 + 인재상 + MBTI/사주 |
| 결과 저장 | `recruitment_reports` 테이블 (report_type: comprehensive) |
| 상태 변경 | → `analyzed` |

**종합 분석 결과 항목:**
- `overall_score`: 종합 점수 (0-100)
- `summary`: 총평
- `detailed_analysis`: 이력서 적합도, 면접 성과, 문화 적합도, 성장 잠재력
- `talent_match`: 회사 인재상 매칭 결과
- `saju_mbti_analysis`: 성격/업무 스타일/팀 호환성 (참고용)
- `salary_recommendation`: 연봉 권장
- `department_recommendation`: 추천 부서
- `position_recommendation`: 추천 직급
- `ai_recommendation`: STRONG_HIRE / HIRE / REVIEW / NO_HIRE
- `key_strengths`: 핵심 강점
- `key_concerns`: 핵심 우려

---

### 11단계: 합격/불합격 결정 (analyzed → hired 또는 rejected)

| 항목 | 내용 |
|------|------|
| 경로 | 지원자 상세 > 의사결정 섹션 |
| 주체 | HR 관리자 / 결정권자 |
| 동작 | "합격" 또는 "불합격" 버튼 |

**합격 시 입력:**
- 제시 연봉 (만원)
- 수습 기간 급여 (선택)
- 정규직 전환 급여 (선택)
- 직무/직급
- 입사 예정일

**자동 처리:**
1. `hiring_decisions` 기록 (offer_token 포함)
2. `candidates.status` → `hired` 또는 `rejected`
3. **AI 신뢰도 자동 기록** — AI 추천 vs 실제 결정 비교 (`ai_accuracy_log`)
4. **신뢰도 지표 업데이트** — `ai_trust_metrics` 자동 집계
5. **이메일 발송**:
   - 합격: `hiringAcceptEmail()` — 축하 + 조건표 + 응답 링크
   - 불합격: `hiringRejectEmail()` — 정중한 안내

---

### 12단계: 합격 조건 응답 (hired 상태 유지)

| 항목 | 내용 |
|------|------|
| 경로 | `/accept/:offer_token` (외부 페이지) |
| 주체 | 합격자 |
| 이메일 링크 | 합격 이메일 내 "합격 조건 확인" 버튼 |

**지원자 응답 항목:**
- 합격 조건 동의 여부 (필수)
- 연봉 협상 의사 (없음 / 협상 희망 + 희망 금액)
- 희망 입사일 변경 (선택)
- 기타 요청사항 (텍스트)

**자동 처리:**
- `hiring_decisions.candidate_response`에 저장
- HR 담당자에게 알림 이메일 발송 (`hr@interohrigin.com`)

**관리자 확인:**
- 지원자 상세 페이지에서 제시 조건 + 지원자 응답 확인 가능
- 미응답 시 "아직 응답하지 않았습니다" 안내

## AI 분석 시점

| 단계 | AI 기능 | 트리거 | 입력 데이터 |
|------|---------|--------|------------|
| 2단계 | 이력서 분석 | "AI 분석" 버튼 | 이력서 + 자기소개서 + 공고 정보 |
| 4단계 | 질의서 포함 재분석 | "재분석" 버튼 (선택) | 위 + 사전질의서 응답 |
| 5단계 | 면접 자동 배정 | "AI 자동 배정" 버튼 | 지원자 목록 + 가용 시간대 |
| 7/9단계 | 면접 분석 | "AI 내용 분석" 버튼 | 녹화/녹음 파일 (STT + 내용 분석) |
| 10단계 | 최종 종합 분석 | "최종 종합 분석" 버튼 | 전체 데이터 통합 |
