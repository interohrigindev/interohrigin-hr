# HR 플랫폼 고도화 개발 계획

## 1. 문서 개요

- 기준일: 2026.04.21
- 범위: 수습평가 보강 + 퇴직자 인수인계 자동화 + 운영 피드백 보완
- 목적: 평가 결과를 성장 피드백 문서로 전환하고, 퇴사 시 조직 지식이 사라지지 않도록 인수인계 자동화를 구축하며, 실제 사용자의 운영 피드백을 기반으로 기존 화면의 막힘과 사용성 문제를 해소
- 전제: 기존 READ ONLY 테이블(`employees`, `evaluations`, `evaluation_items`, `users`)은 ALTER 하지 않음
- 참고 원본:
  - `docs/ref/차대표님_확인요청_정리_20260421.md`
  - `/Users/project.adv/Downloads/개인 페이지 & 공유된 페이지`

## 2. 기대 효과

### 평가 보강
- 평가자(리더/임원)가 짧은 코멘트만 입력해도 AI가 구체적인 성장 피드백으로 확장
- 직원 단위 강점/약점/조언을 누적 분석하여 수습 종료 판단과 육성 방향 정리에 활용
- PDF 1페이지 출력으로 캡처 없이 보고 가능한 평가서 확보
- 평가 카드에 입사일/수습종료일 표시로 판단 근거 명확화

### 인수인계 자동화
- 프로젝트, 업무보고, 자산/계약서, 문서 위치를 기반으로 퇴직자 인수인계 초안 자동 생성
- 후임자가 챗봇으로 계약서/자산/문서 위치를 검색 가능
- 관리자 관점에서 누락 없는 인수인계 체크리스트와 지식 보존 체계 확보

### 운영 피드백 보완
- 직원 계정 기준으로 실제 막히는 평가/월간점검/OJT/멘토링 진입 문제를 해소
- 프로젝트/일일보고/연차/전자결재 화면을 보고용이 아니라 실제 업무용 흐름으로 개선
- 전달된 결재선/인감/평가/캘린더 참고 이미지를 구현 기준으로 반영

## 4. Phase A 상세

### A1. 입사일 + 수습종료일 표시
- 대상 파일: `src/routes/ojt/probation-results.tsx`
- 반영 위치: 평가 카드 `CardHeader`
- 표시 형식: `입사일: YYYY.MM.DD · 수습종료: YYYY.MM.DD`
- 계산 기준: 기존 `probation.tsx`의 `hire_date + 90일` 계산 로직 재사용
- 데이터 요구: `employees.hire_date` 조회 확인, 누락 시 select 보강
- 완료 기준: 평가 카드/상세 다이얼로그에서 동일 정보 노출

### A2. 수습평가 PDF 출력
- 신규 파일: `src/lib/pdf-probation.ts`
- 수정 파일: `src/routes/ojt/probation-results.tsx`
- 재사용 기준: `src/lib/pdf-report.ts`, `src/lib/pdf-fonts.ts`
- 출력 목표: A4 1페이지

PDF 포함 항목:
- 헤더: 회사 로고/인감, 문서명, 생성일
- 직원 정보: 이름, 소속, 입사일, 수습종료일, 평가 단계
- 점수 테이블: 5항목 × 20점, 총점
- 총평/칭찬/보완점/역할별 필드
- 수습 지속 권고 배지
- AI 분석 섹션
- 평가자 서명 영역

완료 기준:
- 한글 폰트 정상 렌더링
- 1페이지 중심 레이아웃
- 카드/다이얼로그 양쪽에서 `PDF 저장` 동작 가능

### A3. 평가 코멘트 AI 확장
- 대상 파일: `src/routes/ojt/probation-results.tsx`
- 현재 `polishText()`를 `mode: 'polish' | 'expand'` 방식으로 확장
- 버튼 구성:
  - `다듬기`: 기존 의미 유지 리라이팅
  - `확장하기`: 조언/예시/실행 제안 포함 3~5문장 피드백 생성
- AI 규칙:
  - 비난형 표현 금지
  - 성장 지원 관점 유지
  - `결정` 대신 `제안/권장` 표현 사용

완료 기준:
- 짧은 코멘트 입력 시 두 모드 결과가 명확히 다르게 출력
- 결과 텍스트를 그대로 저장 가능

### A4. 개인 강점/약점 종합 분석
- 대상 파일: `src/routes/ojt/probation-results.tsx`
- AI 호출 재사용: `src/lib/ai-client.ts`의 `generateAIContent()`
- 입력: 직원별 전체 회차, 평가자별 코멘트, 점수 요약
- 출력 JSON:

```json
{
  "strengths": ["문자열"],
  "weaknesses": ["문자열"],
  "advice": ["문자열"],
  "actionPlan": "문자열"
}
```

- UI 반영:
  - 카드 내부 `AI 종합 분석` 섹션
  - `다시 분석` 버튼

완료 기준:
- 회차가 2건 이상인 직원에서 종합 결과 생성
- 강점/약점/조언이 리스트로 렌더링
- 실행계획 문장이 별도 강조 영역에 표시

- `PDF 저장` 클릭 시 한글 1페이지 출력 확인
- `확장하기` 버튼으로 3~5문장 피드백 생성
- 종합 분석 결과가 JSON 구조로 안정 출력

### Phase B
- 테스트 퇴사자 기준 자동 초안 생성 성공
- 자산 인벤토리 등록/수정/반납 상태 변경 확인
- 챗봇에서 계약서/자산/문서 위치 질의 응답 확인
- `npx tsc -b` 또는 `npm run build` 통과
- Supabase RLS로 관리자/후임/본인 외 차단 확인

### Phase C
- 직원 계정에서 목표 설정 후 평가 진행 가능
- 월간 점검 상단 탭 제거 및 우측 특이사항 입력만으로 처리 가능
- 동료평가가 10항목 × 10점, 총점 100점 구조로 표시
- OJT AI 퀴즈 생성 오류 재현 불가
- 멘토-멘티 화면이 직원 계정에서 노출
- 프로젝트 진행 중 작업이 일일보고 진행 중 작업과 연동
- 연차 캘린더에서 직원별 휴가가 월간 뷰로 확인 가능
- 전자결재 메인에서 품의 종류 진입 구조가 분리되고 작성 영역 가독성이 개선됨

## 11. 확정된 설계 결정

- 인수인계 초안 생성은 관리자 수동 버튼 방식
- 챗봇 범위는 해당 퇴사자 데이터만 사용
- Google Drive API 자동 스캔을 도입하되 결과는 수동 편집 가능
- OAuth 스코프는 `drive.readonly` 또는 `drive.metadata.readonly` 확장 검토
- 운영 피드백은 별도 신규 모듈보다 기존 화면 보완 우선
- 전자결재는 Works Mobile 참고 구조를 UX 레퍼런스로 사용
- 증명서/평가 PDF 인감 자산은 전달받은 이미지 2종을 후보로 검수 후 적용
