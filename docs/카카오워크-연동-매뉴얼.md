# 카카오워크 연동 매뉴얼

> **대상**: 인터오리진 HR 플랫폼에 카카오워크 결재 알림을 연동하려는 관리자
> **소요시간**: 약 15분
> **권한**: HR 시스템 admin / hr_admin / ceo + 카카오워크 워크스페이스 관리자

---

## 한 줄 요약

> **App Key 1개 입력 → "매핑 동기화" 1회 클릭 → 즉시 4채널(in_app + push + email + **카카오워크**) 결재 알림 발송 시작.**

카카오워크 미설정 시 자동으로 다른 3채널만 발송됩니다 (기존 동작에 영향 0).

---

## 사전 준비

### 1. 카카오워크 관리자 콘솔에서 발급받을 정보

| 항목 | 발급 위치 | 비고 |
|------|----------|------|
| **Bot Access Token** (App Key) | 카카오워크 관리자 콘솔 → 봇/앱 → 신규 봇 등록 → 액세스 토큰 발급 | Bearer 토큰. 외부 노출 금지. |
| **봇 이름** (참고용) | 봇 등록 시 지정한 이름 | 예: `HR결재봇` |
| **봇 권한 scope** | 봇 등록 시 부여 | 최소: `messages:send`, `users:read`, `conversations:write` |

> 카카오워크 봇 등록은 워크스페이스 마스터/매니저 권한 필요.

### 2. HR 시스템 직원 이메일이 카카오워크 가입 이메일과 동일해야 함

- 매핑은 **이메일 기준**으로 자동 동작
- HR 시스템의 `employees.email` ↔ 카카오워크 `users.email` 매칭
- 다른 경우 → 동기화 결과에서 "실패 목록" 으로 표시 → 관리자가 수동 처리

---

## 연동 5단계

### Step 1. 토큰 입력

1. HR 플랫폼 로그인 (admin/hr_admin/ceo 권한 계정)
2. 좌측 메뉴 → **시스템 관리 → 알림 채널 설정** 이동
   - URL: `/admin/system/notification-channels`
3. 화면 하단 **카카오워크 (KakaoWork)** 카드 찾기
4. 입력:
   - **Bot Access Token**: 카카오워크에서 발급받은 액세스 토큰 붙여넣기 (시크릿, 마스킹 저장)
   - **봇 표시 이름**: `HR결재봇` (또는 원하는 이름)
   - **카카오워크 채널 활성화** 체크박스 ON

### Step 2. 활성 채널에 카카오워크 추가

- 화면 상단 **활성 채널** 그리드에서 `카카오워크` 카드 클릭 → 활성 표시 (보라색 테두리)

### Step 3. 저장

- 화면 하단 **전체 설정 저장** 버튼 클릭
- "저장 완료" 토스트 확인

### Step 4. 이메일 매핑 동기화

1. 카카오워크 카드 안의 **이메일 매핑 동기화** 버튼 클릭
2. 자동 처리:
   - 전 직원 `employees.email` 조회
   - 각 이메일로 카카오워크 `users.find_by_email` API 호출
   - 매칭 성공 → `employee_kakaowork_map` 테이블에 upsert
   - 매칭 실패 → 화면에 실패 목록 표시
3. 결과 확인:
   - **매칭 N명 / 실패 M명 / 총 P명** 표시
   - 실패 목록 펼치기 → 어떤 직원이 실패했는지 + 사유 확인

### Step 5. 본인에게 테스트 발송

1. **본인에게 테스트 발송** 버튼 클릭
2. 카카오워크 앱에서 `[테스트] 카카오워크 연동 확인` DM 수신 확인
3. 성공 시 즉시 모든 결재 알림이 카카오워크로도 발송 시작

## 동작 흐름

```
결재 송신/단계전환/최종승인/반려 이벤트
         ↓
src/lib/approval-notification.ts (디스패처)
         ↓
4채널 동시 발송 (Promise.allSettled — silent fail)
  ├── in_app  → 헤더 종 인박스 (즉시)
  ├── push    → 브라우저/모바일 푸시
  ├── email   → 결재자 메일함
  └── kakao_work → /api/send-kakaowork
                    ├── notification_channel_configs 조회
                    │   └─ enabled = false 또는 token 없으면 → skip (skipped 로 기록)
                    ├── employee_kakaowork_map 조회
                    │   └─ 매핑 없으면 → skip
                    ├── KakaoWork API: conversations.open
                    └── KakaoWork API: messages.send_by
                        → 결재자 카카오워크 앱에 1:1 DM 도착
```

**핵심**: 카카오워크 발송이 실패해도 다른 3채널은 정상 발송. 결재 액션 자체는 절대 차단되지 않음.

## 트러블슈팅

### Q1. 매핑 실패한 직원 처리 방법

**원인**: HR 시스템 이메일과 카카오워크 가입 이메일이 다름

**해결책 (3가지 중 선택)**:

**A. HR 시스템 이메일을 카카오워크 이메일과 일치시키기**
- `employees.email` 수정 (관리자 화면 또는 SQL)
- 매핑 동기화 재실행

**B. 카카오워크 가입 이메일을 HR 이메일로 변경하도록 직원에게 안내**
- 카카오워크 워크스페이스에서 본인 이메일 변경
- 매핑 동기화 재실행

**C. 수동 매핑 (DB 직접 입력)** — 본인 카카오워크 user_id 알 때
```sql
-- 예: employees.id = 'xxx', 카카오워크 user.id = 12345
INSERT INTO employee_kakaowork_map (employee_id, kakaowork_user_id, email_used, display_name)
VALUES ('xxx', '12345', 'manual@example.com', '홍길동')
ON CONFLICT (employee_id) DO UPDATE
  SET kakaowork_user_id = EXCLUDED.kakaowork_user_id,
      synced_at = now();
```

### Q2. 토큰 입력했는데 테스트 발송 시 "skipped: kakaowork-disabled" 가 뜸

- **카카오워크 채널 활성화** 체크박스가 OFF 상태일 가능성
- "활성 채널" 그리드에서 카카오워크 카드도 클릭해 활성화 했는지 확인
- 저장 버튼 눌렀는지 확인

### Q3. 테스트 발송 시 "skipped: no-mapping" 이 뜸

- 본인 이메일이 매핑되지 않은 상태
- 매핑 동기화 다시 실행 → 본인이 실패 목록에 있는지 확인 → 위 Q1 해결책 적용

### Q4. 결재 알림은 가는데 일부 직원만 못 받음

- 해당 직원의 매핑이 빠진 상태
- 매핑 동기화 재실행 (신규 입사자 등)

### Q5. 카카오워크 채널을 끄고 싶음

- 카카오워크 카드의 **활성화 체크박스 OFF** + 저장
- 즉시 모든 결재 알림에서 카카오워크 발송이 skip 됨
- 다른 3채널은 정상 작동

### Q6. App Key 가 노출되었을 때

- 카카오워크 관리자 콘솔 → 봇 → 액세스 토큰 재발급
- HR 플랫폼 알림 채널 설정 화면에서 새 토큰 입력 + 저장

## 보안 / 운영 메모

- **App Key 저장 위치**: `notification_channel_configs.kakaowork_app_key` (DB) — RLS 로 admin/hr_admin/ceo 만 읽기/쓰기
- **클라이언트 노출 X**: API 호출은 모두 서버측(`functions/api/send-kakaowork.ts`)에서 수행
- **HTTPS 통신**: 카카오워크 API 는 모두 https://api.kakaowork.com/v1
- **Rate limit**: 매핑 동기화는 순차 호출 (직원 수만큼) — 대량 직원 시 수초~수십초 소요 가능
- **감사 로그**: 알림 채널 설정 저장 시 `audit_logs` 에 자동 기록 (`logAudit({ action: 'update', entity: 'notification_channel_config' })`)
