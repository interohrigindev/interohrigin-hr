### 2-1. 채용 / 지원자 관리

| 항목 | 내용 |
|------|------|
| **현재 상태** | 지원자 상세 페이지에 히스토리 표시 기능 존재, 헤드헌터 직접 입력 기능 존재 |
| **우선순위** | 🔴 높음 |
| **예상 일정** | 1차 — 다음 주 내 |

#### 개발 항목

**① 지원자 상세 페이지 정보 보강**
- 현재: 기본 히스토리만 표시
- 변경: 면접 일시, 면접관, 면접 결과, 진행 단계 등 상세 이력 추가
- 목표: 해당 페이지 하나만으로 지원자의 전체 채용 프로세스를 파악할 수 있도록 구성

**② 외부 공유 링크 기능**
- 지원자 상세 페이지를 외부(대표님 등)에 링크로 공유할 수 있는 기능 추가
- 별도 로그인 없이 열람 가능한 읽기 전용 공유 페이지 생성
- 공유 링크 만료 기간 설정 검토

**③ 지원자 정렬 로직 변경**
- 새로운 지원자가 접수된 공고가 상단으로 올라오는 정렬 기능 적용
- 정렬 기준: 최신 지원자 접수 시간 기준 내림차순

**④ 지원자 접수 알림**
- 새 지원자 접수 시 담당자에게 이메일 알림 발송
- 대상: 해당 공고의 채용 담당자

### 2-2. 전자결재 / 업무보고

| 항목 | 내용 |
|------|------|
| **현재 상태** | 결재선 설정 오류 수정 완료, 기본 결재 양식 구현 완료 |
| **우선순위** | 🔴 높음 |
| **예상 일정** | UI 개선 — 이번 주 / 양식 개편 — 다음 주 |

#### 개발 항목

**① 결재 화면 UI 개선 — 웹/모바일 분리**
- 모바일: 기존 모달 방식 유지
- 웹: 별도 페이지 오픈 방식으로 변경하여 전체 화면으로 크게 표시
- 업무보고서, 지출결의서 등 모든 결재 양식에 동일 적용

**② 연장근무 신청서 시간 단위 변경**
- 현재: 시간 단위 선택지가 너무 많음
- 변경: 30분 단위로 변경 (예: 0.5h, 1.0h, 1.5h …)

**③ 지출결의서 양식 전면 개편**
- 필드명 변경: 실무 용어로 수정 (예: "지급 요청 금액")
- 입금 계좌 정보 필드 추가
- 상세 내역 입력 칸 너비 확대
- **상세 내역 본문을 편집용 글쓰기(Rich Text Editor) 형식으로 변경** — 이미지 업로드 기능 포함
  - 배경: 결재자가 첨부파일을 직접 열어보지 않고, 본문 내 이미지로 세금계산서/영수증을 확인하는 실무 패턴 반영
- 증빙 파일 첨부 기능 추가 (경비 청구 시 증빙 필수)
- 첨부파일 이미지 미리보기 기능 추가 (결재자가 파일 열지 않고 확인)
- 참고 이미지: 현재 네이버웍스 지출결의서 양식 기준

**④ 전자서명 법적 효력 확인 (보류)**
- 자체 구현 서명의 법적 효력 불명확 (글로사인·모두사인과 차이)
- 노무사 확인 후 도입 여부 결정
- 기술적 구현은 재직증명서 도장 기능과 동일 수준으로 가능

### 2-3. 수습 / 전기 평가

| 항목 | 내용 |
|------|------|
| **현재 상태** | 수습평가 및 정규직 평가 기본 구조 구현 완료, 사용성 이슈 다수 |
| **우선순위** | 🔴 높음 |
| **예상 일정** | 수습평가 로직 — 다음 주 / 정규직 평가 AI — 검토 후 확정 |

#### 수습평가 기능 업데이트

**① 평가 회차 시각적 명확화**
- 현재 문제: 평가 대상자 목록에서 누가 몇 회차인지, D-day가 얼마나 남았는지 한눈에 파악 불가
- 개선: 현재 활성화된 회차를 블럭/하이라이트로 강조 표시
- D-day 임박 순 상단 정렬 적용

**② 평가 회차 순서 강제 로직**
- 이전 회차 미완료 시 다음 회차 진행 불가하도록 잠금 처리
- 이미 초과된(마감된) 회차: 관리자가 강제 마감(잠금) 처리
- 필요 시 관리자가 수동으로 잠금 해제 가능

**③ 평가 기간 자동 활성화/마감**
- 평가 시작일로부터 7일간만 활성화
- 7일 경과 후 자동 마감 → 수정 불가
- 마감 전 알림 또는 시각적 경고 표시 검토

**④ 평가 오류 수정 방지**
- 현재 사례: 김영석 이사가 이수님의 2회차를 평가해야 하는데 3회차를 평가한 오류 발생
- 위 ②번 로직 적용으로 해결

#### 정규직 평가 항목 기능 업데이트

**⑤ 자기평가 AI 가이드 기능**
- 현재 문제: 빈 텍스트 입력창만 있어 직원들이 작성에 어려움을 느낌
- 개선 방향:
  - 평가 항목 설명 기반으로 정량/정성 목표 예시를 AI가 자동 생성
  - 직원이 예시 중 선택 → 부연 설명 작성 (*필수항목)
  - 예시: "예산 집행 효율성 최적화" 항목 → "재고관리 기록 10건 이상 유지", "인플루언서 3명 확보" 등 행동 지표 중심 예시 제시
- 향후 고도화: 본인 프로젝트 및 월간 점검 내용이 자기평가 베이스 데이터로 자동 연동

**⑥ 평가자 레이아웃 개선**
- 현재: 평가자들이 세로로 길게 나열 → 항목과 점수 매칭이 어려움
- 변경: 한 줄에 3명씩 표시되는 블럭 레이아웃 적용

### 3-3. 자원 예약 관리 (신규)

| 항목 | 내용 |
|------|------|
| **현재 상태** | 신규 개발 |
| **우선순위** | 🟡 중간 |
| **예상 일정** | 다다음 주 |

#### 예약 대상 자원

| 자원 | 단위 | 현재 관리 방식 | 비고 |
|------|------|--------------|------|
| 회의실 (지하 대회의실, 소회의실, 4층 회의실) | 시간 | 별도 캘린더 (과금 이슈로 미사용) | 사진 포함 공간명 표시 |
| 스튜디오 | 시간 | 노션 페이지 (태그 + 수동 확인) | 확인 프로세스 불필요, 시간 겹침만 방지 |
| 법인차량 (아반떼 1대) | 일/시간 | 슬랙 | 사용 빈도 낮음 |
| 공용 노트북 (4대) | 대여/반납 | 비공식 구두 요청 | 분출대장 미운영, 마우스 등 부속품 분실 이슈 |

#### 개발 범위

**① 타임슬롯 예약 시스템**
- 회의실, 스튜디오: 시간 단위 타임슬롯 방식 (중복 예약 자동 차단)
- 스튜디오 확인 프로세스 불필요 → 시간 겹침만 방지하면 됨 (김예슬 담당자 확인 완료)
- 법인차량: 사용 빈도 낮으므로 동일 예약 카테고리로 통합

**② 공용 노트북 대여/반납 관리**
- 대여 신청 → 반납 확인 → 상태 관리
- 부속품(마우스 등) 포함 체크리스트 검토

**③ 회의실명 직관적 표시**
- 현재: "입구동굴", "안쪽동굴" 등 직원이 알아보기 어려운 명칭
- 사진 포함하여 공간을 직관적으로 식별할 수 있도록 표시
