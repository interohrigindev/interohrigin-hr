## 공통 베이스 프롬프트

```text
INTEROHRIGIN HR Platform 저장소에서 법적 리스크 대응 모듈을 구현해줘.

절대 규칙:
- employees, evaluations, evaluation_items, users ALTER 금지
- 기존 패턴/네이밍/색상 유지
- 한국어 UI, 날짜 YYYY.MM.DD, 모바일 반응형 필수
- AI 문구는 결정이 아닌 제안/권장 표현
- 민감 정보는 임원/HR 외 비공개
- Phase 자동 전환 금지, 관리자 수동 승인
- 파일 전체 덮어쓰기 금지, 변경 부분만 수정
- 작업 후 반드시 npx tsc -b 와 npm run build 로 검증

참조 문서:
- docs/DB.md
- docs/ROUTES.md
- docs/CONVENTIONS.md
- docs/AI.md
- docs/HR플랫폼_법적리스크대응_보완개발계획_0520.md

출력 방식:
1. 먼저 구현 범위와 영향 파일을 짧게 정리
2. 바로 코드/SQL 수정 진행
3. 마지막에 변경 파일, 검증 결과, 잔여 리스크를 보고
```

## Prompt 1. P0 감사 로그 + 알림 인프라

```text
INTEROHRIGIN HR Platform 저장소에서 법적 리스크 대응 P0를 구현해줘.

- READ ONLY 테이블 ALTER 없이 동작
- SECURITY DEFINER RPC 중심으로 관리자 액션 감사 기록 구조 마련
- 관리자용 감사 로그 조회 화면의 기본 구조까지 포함

작업 범위:
- supabase/migrations/*
- src/types/database.ts
- src/lib/audit*.ts 또는 동등 유틸
- src/lib/notification*.ts 또는 동등 유틸
- src/routes/admin/* 감사 로그 화면
- 필요 시 Sidebar 연결

세부 요구:
- before/after jsonb 저장 구조
- actor_uid, entity_type, entity_id, action_type, created_at 포함
- 민감 데이터 마스킹이 필요하면 유틸 함수로 분리
- 알림 발송 성공/실패 모두 notification_deliveries 에 로그 저장
- feature_rollouts 로 모듈별 on/off 와 시범 운영 범위 제어 가능하게 설계
```
