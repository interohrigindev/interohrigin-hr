## 1. 채용 관리 시스템 — 시연 및 피드백

### 1-1. 채용 공고 사이트

- 잡코리아 등 외부 채용 사이트에서 **자사 채용 페이지로 연결하는 링크** 구현 완료
- 인터오리진 INC 홈페이지 리뉴얼 시 채용 페이지를 통합하거나 링크로 연결 가능
- **HTTPS 보안 인증서 미적용 문제**: 외부 접속 시 "주의 요함" 경고 발생 → INC 사이트 리뉴얼(도메인 교체) 시 해결 예정
- 잡코리아 공고에서 "즉시 지원" 옵션을 해제하고 **홈페이지 지원 링크만 노출**하는 방식으로 변경 (삼양식품 사례 참고)

### 1-2. 채용 대시보드 & 전형 프로세스 (7단계)

전체 채용 전형을 지원자 상세 페이지 한 블록 안에서 일괄 관리하는 구조로 시연 완료:

| 단계 | 내용 | 비고 |
|------|------|------|
| ① 이력서 접수 | PDF만 업로드 가능 (타 형식 차단) | 파일 형식 제한 완료 |
| ② AI 서류 분석 | 이력서 기반 1차 분석 | 인재상 기준 자동 분석 |
| ③ 사전 질의서 발송 | 등록된 이메일로 자동 발송 | 질의 항목 수정 가능 |
| ④ 재분석 (서류+질의) | 사전 질의 포함 통합 분석 | 인재상·질의서 기반 |
| ⑤ 1차 화상 면접 | 구글 미트 자동 생성·발송 | AI 자동 일정 배정 가능 |
| ⑥ 2차 대면 면접 | 녹음 파일 업로드 → AI 분석 | 향후 구글 드라이브 자동 연동 예정 |
| ⑦ 최종 합격/불합격 | 조건 제시 → 지원자 응답 확인 | 연봉·수습 조건 등 |

### 1-3. 주요 기능 상세

**면접 일정 관리**
- 수동 일정 등록 + AI 자동 배정 (면접관 가용 시간 입력 시 자동 스케줄링)
- 구글 미트 링크 자동 생성 및 이메일 발송
- 주간 캘린더 뷰 제공 (월간 뷰는 UI 구성상 보류)

**AI 분석 기능**
- 이력서·사전 질의서·면접 녹음 등 전 단계 데이터를 종합하여 최종 분석
- 각 단계별 의사결정 블록에 다음 액션 표시
- AI 신뢰도: 현재는 **보조 도구** 수준 → 데이터 축적에 따라 정확도 95% 이상 시 추천/자동화 단계로 전환 검토
- AI 합격/불합격 제안과 실제 결과의 매칭률을 지속 추적

**자료 보관**
- 면접 관련 파일 14일간 보관 후 자동 삭제
- "확인 완료" 버튼 클릭 시 보관 시작 (미클릭 시 서버에 계속 축적)

## 2. 신규 기능 요청 및 개선 사항

### 즉시 반영 예정

| # | 요청 사항 | 상세 |
|---|----------|------|
| 1 | **중복 지원자 알림** | 이름 + 연락처(생년월일) 크로스체크 → "중복 지원자입니다. 전형을 진행하시겠습니까?" 알림 + 과거 지원 이력 표시 |
| 2 | **합격→불합격 변경 기능** | 조건 협의 결렬 시 합격 상태를 불합격으로 변경하는 UI 추가 |
| 3 | **구글 미트 링크 재생성** | 면접 중 접속 문제 시 새 링크를 만들어 재발송하는 기능 |
| 4 | **메일 재발송 기능** | 면접 메일 미수신 시 재발송 버튼 |
| 5 | **사전 질의서 중복 문항 삭제** | 경영지원 직무 질의서 내 생년월일·MBTI·혈액형 중복 제거 |
| 6 | **면접 장소 기본값 설정** | 회사 주소 기본 입력 + 네이버 지도 연결 |
| 7 | **채용 공고 오탈자 수정** | 복지 항목: "오프렌들리 데이" → "프렌들리 데이" |

### 중기 개발 과제

| # | 요청 사항 | 상세 |
|---|----------|------|
| 8 | **슬랙 알림 연동** | 면접 일정 확정 시 면접관에게 슬랙 채널 알림 발송 (슬랙 연결은 완료, 기능 연결 필요) |
| 9 | **합격자 → 사원 자동 등록** | 최종 합격자 정보를 직원 관리 모듈로 자동 이관 (안정화 후 적용) |
| 10 | **구글 드라이브 녹화 자동 연동** | 구글 미트 면접 녹화 → 구글 드라이브 저장 → HR 플랫폼 자동 가져오기 |
| 11 | **불합격자 데이터 보관 정책** | 중복 지원자 확인을 위해 불합격자 데이터 5년 보관 (자동 삭제 불가) |

## 3. 인사·노무 시스템 현황

### 연차 관리
- 연차 촉진 대상 기준 변경 필요: 기존 5개 → **3개 잔여 시 촉진 대상**
- **기본값 설정 필요**: 직원별 연차 개수, 근속 연수별 연차 계산 기준 등 → 인사팀에서 데이터 제공 예정

### 전자결재
- 연차 상신 테스트 중 결재 처리 오류 발생 → 수정 필요
- **결재선 수정 기능**: 현재 고정 → 일부 품의에 대해 예외적 수정 가능하도록 변경 요청

### 급여 시스템
- 급여 계산식 데이터 → 이사님이 수요일까지 전달 예정
- 계산식 입력 후 자동 산출 기능 구현 예정

### 인사 평가
- 평가 기능 자체는 구현 완료 (자기평가 → 리더평가 → 임원평가 다단계)
- **평가 항목 확정 필요**: 기존 안이 유사 유형 반복 → 항목 재구성 논의 필요
- 평가 토글 활성화 시 전 직원에게 자동 노출

## 4. 기타 시스템 기능

### 회의록 녹음
- HR 플랫폼 내 회의 녹음 → 텍스트 변환 기능 구현 (OpenAI Whisper API 활용)

### 메신저
- 사내 메신저 기능 탑재 완료 → 실사용 테스트 필요
- 에러 발견 시 즉시 피드백 요청

### 게시판
- 전사 공개 게시판 구현 완료 → 필요한 카테고리만 남기고 정리 가능

### AI 어시스턴트
- 플랫폼 내장 AI 질의 기능 (ChatGPT 유사) → 플랫폼 맥락 기반 + 외부 정보 검색 가능

## 8. 핵심 합의 사항

1. **채용 시스템은 금요일(4/3)부터 실운영** — 에러 발생 시 즉시 차대표에게 피드백
2. **파견업체/헤드헌터도 동일 시스템 사용** — 지원 링크 공유 → 이력서 업로드 → 이후 단계는 지원자 직접 진행
3. **모든 채용 커뮤니케이션은 이메일 기반** — 웹 문자 대신 이메일 자동 발송·회신으로 통일
4. **인사 평가는 다음 주 화요일 모의 테스트** — 수~목 동안 민지 님이 사전 점검
5. **인사·노무 기본값 데이터 조속 전달 필요** — 연차·급여·결재 유형 등
6. **에러·개선 피드백은 HR 플랫폼 프로젝트 메뉴에 직접 기록** — 차대표가 확인 후 팔로업
