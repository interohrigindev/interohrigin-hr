# KPI 근무 평가

우리 회사의 평가 시스템은 상시 점검을 기반으로 하며, 분기별로 본인-동료-리더-경영진이 참여하는 다면 평가 방식입니다.
※ **설계 원칙 :** 모든 단계는 **[본인 → 리더 → 임원]** 순으로 데이터가 흐르며, 후순위자는 전순위자의 데이터를 반드시 조회할 수 있어야 합니다.

---

## 1. 평가 산출 공식 및 배점

시스템 내에서 최종 점수는 아래 공식에 따라 자동 산출되어야 합니다.

- **최종 점수** = `(업적/역량 점수 × 80%)` + `(동료 다면 평가 × 20%)`
- **세부 항목 (Total 10개)**
    - **업적 평가 (70%) :** 직무별 KPI 7개 항목 (각 10점)
    - **역량 평가 (30%) :** 공통 역량 3개 항목 (각 10점)
    - **다면 평가 (20%) :** 지정된 동료들의 상호 피드백 점수 평균

---

## 2. 모듈별 운영 스케줄

시스템은 월별/분기별로 아래 **3가지 모듈** 중 해당되는 화면을 활성화해야 합니다.

### 📅 분기별 목표 설정

- **해당 월 :** 3월, 6월, 9월, 12월 (차기 분기 대비)
- **특이사항 :** 확정된 목표는 다음 분기 평가 시 '평가 항목'으로 자동 바인딩.

| **단계** | **프로세스** | **대상** | **일정** |
| --- | --- | --- | --- |
| 1단계 | 개인 목표 작성 | 전 직원 | 해당 월 1일~10일 |
| 2단계 | 리더 검토 및 피드백 | 리더 | 해당 월 11일 ~ 15일 |
| 3단계 | 임원 검토 및 피드백 | 담당 임원 | 해당 월 16일 ~ 20일 |
| 4단계 | 대표님 최종 확정 | 대표님(시스템 자동) | 해당 월 21일 ~ 25일 |

### 📅 분기별 종합 평가

- **해당 월 :** 1월, 4월, 7월, 10월
- **특이사항 :** 동료 평가는 인사팀이 사전에 매칭한 대상자만 리스트업.
    
    
    | **단계** | **프로세스** | **노출 데이터** | **일정** |
    | --- | --- | --- | --- |
    | 1단계 | 본인 및 동료 평가 | - | 해당 월 1일~10일 |
    | 2단계 | 리더 평가 | 본인 평가  | 해당 월 11일 ~ 15일 |
    | 3단계 | 임원 평가 | 본인 + 동료 + 리더 | 해당 월 16일 ~ 20일 |
    | 4단계 | 대표님 평가(최종 확정) | 본인 + 동료 + 리더 + 임원 | 해당 월 21일 ~ 25일 |

### 📅 월간 업무 점검

- **해당 월 :** 매월 (단, 3, 6, 9, 12월 제외)
- **특이사항 :** 등급/점수 없이 텍스트 위주의 피드백 기록.

| **단계** | **프로세스** | **일정** |
| --- | --- | --- |
| 1단계 | 본인 작성 | 매월 25일 ~ 말일 |
| 2단계 | 리더 피드백 | 익월 1일 ~ 2일 |
| 3단계 | 임원 피드백 | 익월 2일 ~ 4일 |
| 4단계 | 대표님 피드백(최종 확정) | 익월 4일 ~ 5일 |

## 3. 등급 기준 및 보안 로직

### 🔒 평가 익명성 강화

- **열람 제한 :** 평가의 점수와 코멘트는 **인사부서,** **임원 및 대표님만** 열람 가능합니다.
- **리더 권한 :** 직속 리더는 본인 팀원의 '동료 평가' 세부 내용을 볼 수 없으며, 시스템이 합산한 최종 점수 결과만 확인합니다.

### ✅ 등급 자동 매칭

- **S (100~95) / A (94~85) / B (84~70) / C (69~50) / D (49~0)**
- 최종 합산 점수에 따라 등급이 자동 매칭되되, **최종 확정 권한은 임원/대표님**에게 있습니다.

### ⚠️ 예외 로직

- **신규 입사자 :** 재직 기간 1개월 미만일 경우 시스템에서 해당 분기 평가 대상 자동 제외 처리.
- **기한 엄수 :** 정해진 기한이 종료되면 시스템 입력창은 자동 비활성화(Lock) 처리됩니다. (별도 수정 권한 없음/관리자만 수정 권한 있음)

## 4. UI/UX 필수 반영 사항

### **① 매월 점검 : 특이사항 말머리 선택 기능**

- **기능 :** 매월 점검 시 작성하는 '특이사항' 입력창 상단에 아래 4가지 말머리를 드롭다운(선택형)으로 제공합니다.
- **말머리 종류 :** `[이슈]`, `[칭찬]`, `[제안]`, `[기타]`
- **UX 가이드 :** 사용자가 직접 타이핑하지 않고, 선택 시 자동으로 본문 앞머리에 태그 형태로 붙거나 카테고리가 지정되어야 합니다. (추후 대표님/임원 검토 시 카테고리별 필터링 기능 제공 목적)

### **② 데이터 검증: 정수 기반 점수 입력**

- **입력 제한 :** 모든 점수 입력 필드는 정수로만 작성 가능하며, **소수점 입력은 차단**합니다.

### **③ 동료 평가 익명성 및 보안**

- **권한 제한 :** 본인과 직속 리더는 동료 평가의 상세 코멘트와 작성자를 절대 확인할 수 없습니다.
- **노출 범위 :** 동료 평가의 데이터는 오직 **인사부서, 관리자, 임원 및 대표이사**에게만 전체 공개되며, 시스템은 리더에게 합산 점수 결과값만 제공합니다.

### **④ 데이터 아카이빙 및 참조 뷰**

- **화면 구성 :** 분기 종합 평가 화면 구성 시, 해당 분기(1~2월 등)에 작성했던 **'매월 점검'** 텍스트 데이터가 화면 우측 또는 하단에 참조 데이터(Read-only)로 항상 노출되어야 합니다.
- **목적 :** 과거 기록을 보며 현재 성과를 객관적으로 평가할 수 있는 근거를 제공합니다.

### **⑤ 자동 알림 시스템**

- **사전 알림 :** 각 평가/점검 단계 시작 **3일 전**에 미리 알림을 발송합니다.
- **독촉 알림 :** 종료일까지 작성이 완료되지 않은 미완료자에게는 **마감 당일 오전**에 자동 알림을 발송합니다.
