## 1. 개요

본 문서는 인터오리진 측에서 전달한 *"HR AI 플랫폼 고도화 개발 요구서 v1.0"* 의 모든 FR(Functional Requirement) 을 현재 코드베이스 상태와 대조 분석하여 개발 항목 · 우선순위 · 일정 · 리스크를 정리한 실행 계획서입니다.

요구서가 다루는 핵심 영역:
1. **사전 질의서 UI/UX 개선** (응답률·데이터 품질 향상)
2. **채용 프로세스 자동화** (1차 합격 → 적성검사 + 일정 자동 발송, AI 보조 면접 화면)
3. **AI 기반 디자인 플랫폼** (레퍼런스 아카이빙, 비포·애프터 비교, AI 이슈 추천)

## 3. Phase 1 — 사전 질의서 UI/UX 개선

**기한:** 1주 (P0)
**핵심 효과:** 응답률 +30% 예상 (요구서 기재)

### 3-1. 작업 항목

#### A. 단계형 질문 UI (FR-101)

**파일:** [src/routes/public/survey.tsx](src/routes/public/survey.tsx) 재구성

```tsx
// 새 상태 구조
const [currentStep, setCurrentStep] = useState(0)  // 0=기본정보, 1+=질문
const totalSteps = 1 + surveyQuestions.length + personalityItems.length
const progress = Math.round(((currentStep) / (totalSteps - 1)) * 100)
```

**UI 요소:**
- 상단 고정 진행률 바 (`<div className="h-1 bg-brand transition-all" style={{ width: progress + '%' }} />`)
- "3 / 10 완료" 텍스트
- "다음" 버튼 — 현재 질문 응답 검증 후 다음 스텝으로
- "이전" 버튼 — 직전 답변 복원 (state 유지)
- 마지막 스텝에 "제출" 버튼

#### B. 기본정보 분리 (FR-102)

- Step 0 전용 화면: 이름·생년월일·MBTI·혈액형·한자이름 등 메타데이터만
- Step 1+: 순차 질문
- 기존 `pre_survey_data.meta` / `pre_survey_data.answers` 구조 유지 (DB 호환)

#### C. 적성진단 문항 자연 삽입 (FR-103)

**신규 테이블 (migration 073):**
```sql
CREATE TABLE personality_test_items (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  category text NOT NULL, -- 'EI','SN','TF','JP' (MBTI) or 'spectiv'
  question text NOT NULL,
  positive_dimension text, -- 답변이 yes 일 때 어느 차원 가중
  display_order int DEFAULT 0,
  is_active boolean DEFAULT true,
  created_at timestamptz DEFAULT now()
);
```

**삽입 위치:**
- 면접용 질문 사이에 무작위로 분산 (`shuffleWithInterleave([surveyQs, personalityQs])`)
- 지원자에게 "적성검사" 라는 라벨을 별도로 노출하지 않음 — 일반 질문처럼 보이도록

**응답 저장:**
- `candidates.personality_test_result jsonb` 컬럼 추가
- 차원별 점수 합산 → 결과 도출 ("ENFP 70%" 식)

#### D. 자동 저장 (FR-104)

**이중 저장 전략:**
1. **localStorage** (즉시): `survey-draft-{token}` 키에 매 응답 시 저장 (0~500ms debounce)
2. **DB draft** (5초 debounce): `candidates.pre_survey_draft jsonb` 컬럼에 저장

페이지 재진입 시:
```ts
const draft = localStorage.getItem(`survey-draft-${token}`)
  || (await supabase.from('candidates').select('pre_survey_draft').eq('invite_token', token).single()).data?.pre_survey_draft
if (draft) restoreAnswers(draft)
```

제출 완료 시 draft 모두 삭제.

#### E. 제출 결과 (FR-105)

- 기존 AI 1차 분석에 적성 진단 결과를 추가 컨텍스트로 전달
- 관리자 화면에서 "적성 진단 결과" 별도 박스 노출 (지원자에게는 미공개)

## 5. Phase 3 — AI 기반 디자인 레퍼런스 플랫폼

**기한:** 3~4주 (P2)
**위치:** 신규 모듈 — `/admin/design-references/*`

### 5-1. DB 설계 (migration 078)

```sql
CREATE TABLE design_references (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  title text NOT NULL,
  purpose text, -- 자동 추천될 태그 (광고/마케팅/디자인 등)
  category text, -- 카테고리 분류
  files jsonb DEFAULT '[]'::jsonb, -- [{url, mime_type, size}]
  links jsonb DEFAULT '[]'::jsonb, -- [{url, label}]
  body_text text, -- 메모/캡션
  shared_with jsonb DEFAULT '[]'::jsonb, -- department_id 또는 employee_id 배열
  uploaded_by uuid REFERENCES employees(id),
  ai_suggestions jsonb, -- AI 생성 카피·디자인 이슈
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);

CREATE TABLE design_reference_outputs (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  reference_id uuid REFERENCES design_references(id) ON DELETE CASCADE,
  output_files jsonb NOT NULL, -- 결과물 파일
  comparison_notes text,
  ai_similarity_score int, -- 0~100
  ai_feedback jsonb,
  uploaded_by uuid REFERENCES employees(id),
  created_at timestamptz DEFAULT now()
);

CREATE TABLE design_reference_comments (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  output_id uuid REFERENCES design_reference_outputs(id) ON DELETE CASCADE,
  author_id uuid REFERENCES employees(id),
  content text NOT NULL,
  anchor jsonb, -- {x, y} 좌표 어노테이션 (선택)
  resolved boolean DEFAULT false,
  created_at timestamptz DEFAULT now()
);
```

### 5-2. 라우트 구조

- `/admin/design-references` — 목록 (그리드 카드 뷰)
- `/admin/design-references/:id` — 상세 (레퍼런스 + AI 추천 + 결과물 비교)
- `/admin/design-references/:id/compare/:outputId` — 비포·애프터 분할 비교 화면

### 5-3. AI 추천 (FR-302)

- 업로드 후 자동으로 Gemini multimodal 호출
- 입력: 이미지 파일들 + 사용자 입력 메모(목적/스타일)
- 출력 JSON:
  ```json
  {
    "copy_suggestions": ["광고 카피 5종"],
    "design_issues": ["디자인 가이드 3가지"],
    "ai_correction_prompts": ["AI 수정 명령어 추천"]
  }
  ```
- 이미지 토큰 비용이 크므로 — 1회만 자동 호출, 재생성은 사용자 명시 요청 시

### 5-4. 비포·애프터 비교 (FR-303)

- React Compare Slider 또는 좌/우 50:50 분할
- 우측 결과물에 코멘트 핀(좌표 + 텍스트) 가능
- AI 유사도 점수 자동 계산 (선택 — 비용 부담 시 수동)

### 5-5. 비용 가드

- **Storage 정책:** 레퍼런스 파일은 90일 후 자동 압축(웹용 해상도 다운스케일) — 비용 절감
- **AI 호출 정책:** 업로드 시 1회만 자동, 재생성은 명시적 클릭
- **권한:** 부서별 share — 전체 노출 방지

## 8. 리스크와 대응

| 리스크 | 영향도 | 대응 |
|--------|--------|------|
| 적성검사 문항 출처 (Spectiv 라이선스) | 중 | 자체 30문항 셋 작성 또는 MBTI 풀 사용. 운영팀 확정 필요 |
| Realtime 동시 입력 정합성 (Phase 2-3) | 중 | last-write-wins + 충돌 감지 토스트 |
| 이미지 레퍼런스 스토리지 비용 | 고 | 90일 자동 압축 + 사용량 모니터링 + 부서별 쿼터 검토 |
| 슬롯 선택 — 동시 예약 충돌 | 중 | 예약 시 `SELECT FOR UPDATE` 또는 Supabase RPC + 트랜잭션 |
| AI 호출 비용 (Gemini multimodal) | 고 | 자동 호출 1회 제한, 재생성은 명시 클릭, 캐시 활용 |
| 모바일 UX (캘린더 슬롯 / 디자인 비교) | 중 | 375px 모바일 우선 디자인. 데스크톱은 확장 |

## 9. 미결 논의 사항

- [ ] **적성 검사 문항 출처** — Spectiv 라이선스 확보 vs. 자체 풀 vs. MBTI
- [ ] **사전질의서 예상 소요 시간 10분** — 현재 문항 수 측정, 단계형 전환 후 재측정

- [ ] **레퍼런스 아카이빙 용량 정책** — 무제한 vs. 부서별 1GB 등
- [ ] **AI 디자인 추천 PoC 결과 검토** — 비용 vs. 효용 합의
- [ ] **합격 통보 SMS 도입 여부** — Aligo 등 한국 SMS 게이트웨이 비교
- [ ] **면접 콘솔 다인 동시 입력 시 권한 모델** — 면접관 명단 외 비공개 여부
